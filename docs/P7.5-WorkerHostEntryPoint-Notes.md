# P7.5 Worker Host Entry Point Notes

This set adds an executable worker host project that wires the P7 SQL operational runtime into a process that can be started locally or deployed as a cloud worker.

The included executor is intentionally a smoke executor. It completes claimed work items and writes a JSON result marker. This lets the platform prove the SQL queue, hosted loop, leasing, completion, and readiness path before connecting real migration connector behavior.

## Important runtime behavior

- On startup, the host opens the configured SQL connection and runs the readiness probe.
- If the probe fails, the process fails fast.
- If the probe passes, the hosted SQL operational worker starts.
- The worker claims batches from SQL using the P7.1/P7.2 queue procedures.
- Claimed work is completed by the smoke executor.
- `RunUntilIdleAndStop` can be used for local validation and CI smoke execution.

## Package management

The worker project has no inline package versions. If `Microsoft.Data.SqlClient` is not already centrally versioned, add it to `Directory.Packages.props` using the repo's existing central package management pattern.
