# P10.2AD — Admin Web Execution Sessions Feature Move

## Purpose

Continue canonical Admin Web consolidation without drift. The deployable Admin UI remains:

`src/Admin/Migration.Admin.Web`

The `/apps/migration-admin-ui` tree remains reference-only.

## Scope

This set moves one operations feature family: Execution Sessions.

The move is intentionally narrow:

- move the page, API client, and type contract into `features/operations/executionSessions`
- update the `App.tsx` import for `ExecutionSessions`
- keep existing routes intact
- avoid broad route patching
- avoid mass folder moves

## Safety notes

The scripts are written for Windows PowerShell 5.1 and `Set-StrictMode -Version 2.0`.

They avoid:

- nested validation arrays that PowerShell flattens
- self-recursive validator scans
- brittle `\bin\` / `\obj\` regex checks
- backslash escape sequences in generated path strings
- reading files before `Test-Path -PathType Leaf`
