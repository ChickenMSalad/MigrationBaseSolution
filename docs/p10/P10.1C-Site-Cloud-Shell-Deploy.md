# P10.1C — Site Cloud Shell Deploy

## Goal

Deploy the repo-native admin site/API shell to Azure without changing runtime worker behavior.

## Repo evidence

The current site/cloud host candidate is:

```text
src/Core/Migration.Admin.Api/Migration.Admin.Api.csproj
```

The Vite web shell exists at:

```text
src/Admin/Migration.Admin.Web
```

but the current repo bundle only showed `package.json`, `package-lock.json`, and `tsconfig.json` there. Until the UI source tree is restored/confirmed, P10.1C deploys the Admin API shell first. That gives us a real cloud site endpoint with Swagger and runtime DB wiring.

## Expected cloud checks

After deployment, the Admin API App Service should:

- start successfully,
- expose Swagger JSON at `/swagger/v1/swagger.json`,
- expose Swagger UI at `/swagger`,
- bind the canonical runtime SQL connection string,
- have no `MIGRATION_*` runtime duplicates required for startup.

## Deployment posture

This set does not mutate project files. It generates a reviewable deployment command script and a readiness probe command.
