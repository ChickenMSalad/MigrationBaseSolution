# P10.2AA - Admin Web Canonical Navigation and Feature Structure Review

## Purpose

P10.2AA is a corrective consolidation gate for the canonical Admin Web application.

The canonical deployable Admin UI remains:

```text
src/Admin/Migration.Admin.Web
```

The feature-source/reference UI remains:

```text
apps/migration-admin-ui
```

This set does not move or delete feature code. It produces a focused report that identifies:

- duplicate navigation entries in the canonical Admin Web layout;
- migrated pages that are still in flat `src/pages`;
- migrated API clients that are still in flat `src/api`;
- migrated type files that are still in flat `src/types`;
- recommended feature-folder destinations under `src/features`.

## Why this exists

P10.2D through P10.2Z moved many operational UI surfaces into the canonical Admin Web, but most were intentionally added as flat files first to keep build risk low.

This set is the checkpoint before reorganizing those files into feature folders.

## Expected next step

After this report is reviewed, the next implementation set should move one feature family at a time into the canonical feature folders, starting with operations/runtime dashboard files.
