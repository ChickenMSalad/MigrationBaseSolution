# P10.1B — Site Cloud AppSettings Wiring

## Purpose

Wire the existing `Migration.Admin.Api` site/API host to the stabilized cloud runtime without changing runtime worker behavior.

This set is intentionally narrow. It does not deploy the site and it does not mutate Azure settings automatically. It defines the canonical cloud settings contract and provides command generation/validation tooling so the Admin API can be configured consistently before P10.1C deployment.

## Repo evidence used

The active site/API host is:

```text
src/Core/Migration.Admin.Api/Migration.Admin.Api.csproj
```

Its `Program.cs` wires runtime-facing services and endpoints including:

```text
AddAdminApiOperationalRuntimeReadiness
AddAdminApiOperationalRunCoordinator
AddSqlOperationalWorkItemQueue
AddAdminApiSqlOperationalBackbone
AddMigrationAdminApiRuntime
AddOperationalStore
AddMigrationAdminApiOperationalRunMirror
AddMigrationAdminApiAuthentication
AddOperationalAdminServices
MapSqlOperationalWorkItemQueueEndpoints
MapSqlOperationalRunCoordinatorEndpoints
MapSqlOperationalRuntimeReadinessEndpoints
MapMigrationOperationalEndpoints
```

The repo `appsettings.json` currently defines only local/default Admin API and migration execution sections, so Azure cloud wiring should be supplied through App Service settings rather than by editing local defaults.

## Canonical cloud settings

Required Azure App Service settings for the Admin API/site host:

```text
ConnectionStrings__MigrationOperationalStore
SqlOperationalRuntimeReadiness__ConnectionString
SqlOperationalWorkItemQueue__SchemaName
SqlOperationalWorkItemQueue__WorkItemsTableName
AdminApi__StorageRoot
AdminApi__AllowInProcessExecution
MigrationExecution__StatePath
MigrationExecution__ResumeCompletedWorkItems
MigrationExecution__MaxDegreeOfParallelism
MigrationExecution__StopOnFirstError
MigrationExecution__ContinueOnValidationError
```

For cloud safety, keep:

```text
AdminApi__AllowInProcessExecution=false
```

The site should observe and coordinate the runtime first. In-process execution should not be enabled in Azure unless explicitly reintroduced as a controlled feature.

## Usage

Generate Azure CLI commands from the sample after filling placeholders or after creating an environment-specific copy:

```powershell
.\tools\runtime\New-P101SiteCloudAppSettingsCommands.ps1 `
  -SettingsTemplatePath .\config-samples\p10-site-cloud-appsettings.canonical.azure.sample.json `
  -ResourceGroup $resourceGroup `
  -AppName $siteApp `
  -OutputPath .\artifacts\site\p10-site-appsettings-apply.ps1
```

Validate exported live app settings:

```powershell
az webapp config appsettings list `
  --resource-group $resourceGroup `
  --name $siteApp `
  -o json > .\artifacts\site\site-live-settings.json

.\tools\runtime\Test-P101SiteCloudAppSettingsContract.ps1 `
  -SettingsPath .\artifacts\site\site-live-settings.json
```

## Exit criteria

```text
Admin API/site cloud appsettings are generated from canonical keys
required runtime SQL keys exist
required work-item queue keys exist
in-process execution is explicitly disabled
no MIGRATION_* runtime aliases are required for site startup
```
