# P10.2AV - Admin Web Residual Feature Asset Consolidation

## Purpose

After the batched page/component parity work, the canonical Admin Web can still contain flat feature-related assets under:

- `src/Admin/Migration.Admin.Web/src/api`
- `src/Admin/Migration.Admin.Web/src/types`

This set consolidates those assets into matching existing feature folders under:

- `src/Admin/Migration.Admin.Web/src/features`

## Safety model

- Local-state driven.
- Does not assume failed repairs were committed.
- Does not overwrite existing destination files.
- Does not move non-feature core API files.
- Does not patch routes.
- Rewrites only import source paths inside files that are already in feature folders.
- Writes an evidence report under `artifacts/p10/P10.2AV/`.
