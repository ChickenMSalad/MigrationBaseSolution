# P10.1A - Site Cloud Readiness Inventory

## Purpose

This set starts the site-up path without changing runtime behavior. It records the web/admin host candidates, expected cloud wiring, and evidence needed before deploying the site against the stabilized runtime.

## Repo-grounded direction

The repository root identifies the frontend at `apps/migration-admin-ui` and describes the Control Plane as an ASP.NET Minimal API plus React UI. It also lists the primary UI routes for projects, runs, artifacts, credentials, connectors, mapping builder, and preflight. P10.1 should therefore focus on the existing admin UI/API path rather than inventing a new site host.

## Candidate components

| Component | Repo path | Role |
| --- | --- | --- |
| React admin UI | `apps/migration-admin-ui` | User-facing site shell |
| Admin API | `src/Core/Migration.Admin.Api` | Control Plane API and runtime data access |
| Runtime SQL | `migration.WorkItems`, `migration.ManifestRows`, `migration.Runs` | Cloud runtime state |
| Runtime workers | `src/Workers/Migration.Workers.ServiceBusDispatcher`, `src/Workers/Migration.Workers.ServiceBusExecutor` | Execution plane already stabilized |

## P10.1A exit criteria

- The repo has a documented site/API cloud target.
- The expected cloud appsettings contract is captured.
- A bounded validator confirms the expected UI/API paths and config sample exist.
- No runtime code is mutated.

## P10.1 next steps

P10.1B should normalize appsettings for the Admin API and UI deployment target. P10.1C should publish/deploy the site shell and prove it starts against the cloud runtime.
