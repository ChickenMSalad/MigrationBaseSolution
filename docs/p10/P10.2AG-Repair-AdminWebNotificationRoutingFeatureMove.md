# P10.2AG Repair — Admin Web Notification Routing Feature Move

## Reason

The original P10.2AG apply script failed to parse in Windows PowerShell because it used interpolated strings containing `$Label:`. In PowerShell, that can be parsed as a scoped variable reference and fails before execution.

## Scope

This repair does not advance to a new feature. It completes the intended P10.2AG move for Notification Routing only.

## Guardrails

- PowerShell 5.1 compatible.
- `Set-StrictMode -Version 2.0` safe.
- Uses `[pscustomobject]` records for validation data.
- Does not use nested array records that PowerShell can flatten.
- Validates source or destination for every file before moving anything.
- Uses `Join-Path` instead of fragile backslash escape strings.
- Does not recursively scan validator scripts or flag itself.
