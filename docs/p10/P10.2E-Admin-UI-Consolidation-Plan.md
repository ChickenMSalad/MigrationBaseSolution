# P10.2E - Admin UI Consolidation Plan

## Purpose

P10.2E corrects the Admin UI drift by declaring `src/Admin/Migration.Admin.Web` as the canonical Admin UI and treating `apps/migration-admin-ui` as a feature source/prototype area until useful features are migrated.

This set does not move or delete feature code. It creates a bounded consolidation inventory and guardrail so future P10 Admin UI work targets the canonical site.

## Current state

Canonical site:

```text
src/Admin/Migration.Admin.Web
```

Feature source/prototype:

```text
apps/migration-admin-ui
```

The runtime/API work completed through P10.2D remains anchored in the canonical site. P10.2E pauses further dashboard feature expansion until the feature families from `apps/migration-admin-ui` are intentionally mapped into `src/Admin/Migration.Admin.Web`.

## Consolidation rule

New Admin UI feature work must go under:

```text
src/Admin/Migration.Admin.Web
```

The `apps/migration-admin-ui` app may be used only as a reference/source for migration until it is quarantined or removed.

## Feature migration order

1. Runtime dashboard and run detail
2. Execution sessions and work items
3. Failure retry workspace
4. Manifest import / run launch
5. Credential vault
6. Connector configuration
7. Operational events / audit trail
8. Worker telemetry
9. Command center / capacity / cost views

## Exit criteria

- A consolidation inventory exists.
- The canonical UI is documented.
- The prototype app is marked as non-canonical.
- Validators fail if this set loses the canonical-path decision.
