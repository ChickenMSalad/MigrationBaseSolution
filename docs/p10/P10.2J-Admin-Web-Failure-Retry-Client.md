# P10.2J - Admin Web Failure / Retry Client

## Purpose

P10.2J migrates the next operator-critical surface from the feature-source UI into the canonical Admin Web app.

Canonical UI:

```text
src/Admin/Migration.Admin.Web
```

Feature-source/reference UI:

```text
apps/migration-admin-ui
```

This set adds the canonical Admin Web client/page for failure and retry visibility. It intentionally does **not** mutate the legacy feature-source app and does **not** make `apps/migration-admin-ui` deployable.

## Scope

Added to canonical Admin Web:

```text
src/Admin/Migration.Admin.Web/src/types/failureRetry.ts
src/Admin/Migration.Admin.Web/src/api/failureRetryApi.ts
src/Admin/Migration.Admin.Web/src/pages/FailureRetry.tsx
```

## Route wiring

This set does not patch `App.tsx`. Previous route patching caused drift when the committed one-line/minified React file shape changed. The next route-only set should wire this page once the current committed `App.tsx` is read directly from the repo.

Expected page import when route wiring is added:

```ts
import { FailureRetry } from "./pages/FailureRetry";
```

Suggested route path:

```text
/runtime/failures
```
