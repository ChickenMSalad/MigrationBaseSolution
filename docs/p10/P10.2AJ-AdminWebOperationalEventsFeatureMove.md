# P10.2AJ - Admin Web Operational Events Feature Move

## Intent

Continue P10.2 Admin UI consolidation by moving one remaining operations feature family into the canonical feature folder structure.

## Scope

Canonical Admin Web only:

- `src/Admin/Migration.Admin.Web/src/pages/OperationalEvents.tsx`
- `src/Admin/Migration.Admin.Web/src/api/operationalEventsApi.ts`
- `src/Admin/Migration.Admin.Web/src/types/operationalEvents.ts`

Target:

- `src/Admin/Migration.Admin.Web/src/features/operations/operationalEvents/pages/OperationalEvents.tsx`
- `src/Admin/Migration.Admin.Web/src/features/operations/operationalEvents/api/operationalEventsApi.ts`
- `src/Admin/Migration.Admin.Web/src/features/operations/operationalEvents/types/operationalEvents.ts`

The apply script updates local imports inside the moved page/API files and updates `App.tsx` to import the page from the feature folder.

## Safety notes

- Validates every required source before moving anything.
- Idempotent when the feature has already been moved.
- PowerShell 5.1 compatible.
- `Set-StrictMode -Version 2.0` safe.
- Uses object validation data, not nested arrays.
- Avoids recursive self-scanning and `$variable:` interpolation.
