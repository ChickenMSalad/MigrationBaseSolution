# P10.2O — Admin Web Connector Configuration Route

## Purpose

Wire the already-migrated `ConnectorConfiguration` page into the canonical Admin Web application.

This set keeps the canonical UI decision intact:

- Canonical Admin UI: `src/Admin/Migration.Admin.Web`
- Feature-source/reference UI: `apps/migration-admin-ui`

## Scope

This set only adds route/navigation wiring for the existing canonical page:

- `src/Admin/Migration.Admin.Web/src/pages/ConnectorConfiguration.tsx`

It does not make `apps/migration-admin-ui` deployable and does not copy additional feature-source files.

## Expected result

After applying the route wiring, Admin Web exposes:

- `/connector-configuration`

and the navigation includes:

- `Connector Config`

## Apply

```powershell
.\tools\runtime\Apply-P102AdminWebConnectorConfigurationRoute.ps1
.\tools\validate-p10.2o-admin-web-connector-configuration-route.ps1
```

Then rebuild Admin Web.
