# P10.2AI — Admin Web Command Center Feature Move

## Purpose

Continue Admin Web consolidation by moving the Command Center feature from the flat canonical Admin Web folders into the canonical feature structure.

## Scope

Moves, when present:

- `src/Admin/Migration.Admin.Web/src/pages/CommandCenter.tsx`
- `src/Admin/Migration.Admin.Web/src/api/commandCenterApi.ts`
- `src/Admin/Migration.Admin.Web/src/types/commandCenter.ts`

Into:

- `src/Admin/Migration.Admin.Web/src/features/operations/commandCenter/pages/CommandCenter.tsx`
- `src/Admin/Migration.Admin.Web/src/features/operations/commandCenter/api/commandCenterApi.ts`
- `src/Admin/Migration.Admin.Web/src/features/operations/commandCenter/types/commandCenter.ts`

Also updates imports in:

- `src/Admin/Migration.Admin.Web/src/App.tsx`
- the moved Command Center page
- the moved Command Center API client

## Guardrails

- PowerShell 5.1 compatible.
- `Set-StrictMode -Version 2.0` safe.
- Validates all required source/destination state before moving anything.
- Does not use nested array validation records.
- Does not recursively scan or flag its own validator script.
- Avoids `$variable:` interpolation.
- Uses `Join-Path` instead of backslash-heavy escaped strings.
