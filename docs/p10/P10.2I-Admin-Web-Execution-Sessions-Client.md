# P10.2I - Admin Web Execution Sessions Client

## Purpose

P10.2I begins the controlled migration of the execution-session operator surface from the feature-source app at `apps/migration-admin-ui` into the canonical Admin Web application at `src/Admin/Migration.Admin.Web`.

This set is intentionally additive. It does not modify `App.tsx`, route tables, layouts, package dependencies, or the feature-source app. Route wiring will be a separate set after the canonical page and API client compile cleanly.

## Canonical direction

- Canonical deployable Admin UI: `src/Admin/Migration.Admin.Web`
- Feature-source reference UI: `apps/migration-admin-ui`
- Migrated feature family: execution sessions

## Added files

- `src/Admin/Migration.Admin.Web/src/types/executionSessions.ts`
- `src/Admin/Migration.Admin.Web/src/api/executionSessionsApi.ts`
- `src/Admin/Migration.Admin.Web/src/pages/ExecutionSessions.tsx`

## Follow-up

P10.2J should wire `ExecutionSessions.tsx` into the canonical Admin Web route/navigation after this set builds successfully.
