# P10.2AU - Admin Web Apps Residual Parity Reconcile

## Purpose

After P10.2AP/AQ/AR/AS/AT, the goal is to speed up Admin Web consolidation by reconciling remaining source parity between the historical reference UI and the canonical deployable Admin Web.

Canonical deployable UI remains:

```text
src/Admin/Migration.Admin.Web
```

Reference-only UI remains:

```text
apps/migration-admin-ui
```

## Scope

This set copies only missing files from the reference UI into canonical Admin Web for:

```text
features
components
auth
lib
```

It does not overwrite canonical files, does not mutate route wiring, and does not remove `/apps`.

## Output

The apply script writes:

```text
docs/P10/P10.2AU-AdminWebAppsResidualParityReconcile.Report.md
```

The report lists copied files, skipped existing files, and source roots not present.
