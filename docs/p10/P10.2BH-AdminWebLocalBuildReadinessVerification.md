# P10.2BH - Admin Web Local Build Readiness Verification

Purpose: create a local-state readiness report after the Admin Web consolidation batches. This package intentionally avoids source rewrites and fragile formatter assumptions.

Checks performed:

- Canonical Admin Web source path exists.
- App.tsx exists and can be read from the local working tree.
- App.tsx import sources are inventoried.
- Relative import targets are checked for matching `.ts`, `.tsx`, `.js`, `.jsx`, and `index.*` files.
- Route path strings are inventoried.
- Canonical flat folders are inventoried.
- `/apps/migration-admin-ui` source presence is recorded.
- Admin Web package.json scripts are recorded when available.

The generated report is:

`docs/P10/P10.2BH-AdminWebLocalBuildReadinessVerification.Report.md`
