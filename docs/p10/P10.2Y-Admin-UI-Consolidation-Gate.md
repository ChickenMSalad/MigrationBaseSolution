# P10.2Y - Admin UI Consolidation Gate

## Purpose

P10.2Y pauses feature migration after P10.2X and creates a consolidation checkpoint before additional UI work continues.

The canonical deployable Admin UI is:

```text
src/Admin/Migration.Admin.Web
```

The previous feature-source UI is:

```text
apps/migration-admin-ui
```

This set does not move files. It inventories the canonical Admin Web, the feature-source app, and the migrated P10.2 feature surfaces so the next work can improve structure intentionally instead of continuing to add flat pages indefinitely.

## Current decision

- Keep `src/Admin/Migration.Admin.Web` as the only canonical deployable Admin UI.
- Keep `apps/migration-admin-ui` as feature-source/reference until all useful feature families are migrated.
- Do not add new deployable UI features under `apps/migration-admin-ui`.
- Prefer future canonical grouping under `src/Admin/Migration.Admin.Web/src/features` once the migration report identifies stable feature families.

## Why this matters

The source app has a better feature-folder structure, while the canonical app currently has many files under `src/pages`, `src/api`, and `src/types`. The next consolidation pass should preserve the working build while gradually grouping related files into canonical feature folders.
