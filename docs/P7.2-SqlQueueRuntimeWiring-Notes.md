# P7.2 SQL Queue Runtime Wiring Notes

## What this set adds

P7.2 turns the P7.1 operational store into an executable durable queue path.

It adds:

- `migration.usp_EnqueueManifestWorkItems`
- `migration.usp_StartMigrationRun`
- `migration.usp_CompleteMigrationRunIfDrained`
- `migration.usp_GetRunOperationalSummary`
- `migration.usp_GetRunnableMigrationRuns`
- a dependency-light `SqlOperationalRunStore`
- a dependency-light `SqlOperationalQueueRuntime`
- runtime models/delegate contracts for executing claimed SQL work items

## Why no project file mutation

This set is intentionally conservative. It does not edit `.csproj`, `Directory.Packages.props`, appsettings, or DI registration files. The next set should inspect the current repo surface and wire these types into the actual P6 runtime composition.

## Runtime behavior

The runtime loop does this:

1. claim a batch through P7.1 `migration.usp_ClaimWorkItems`
2. pass each work item to an executor delegate
3. complete successful work through `migration.usp_CompleteWorkItem`
4. fail unsuccessful work through `migration.usp_FailWorkItem`
5. preserve retry/dead-letter behavior in SQL
