# P4.2 — SQL Operational Admin API Facade

## Purpose

P4.2 exposes the P4.1 SQL operational backbone through a narrow Admin API facade.

This set is intentionally additive and conservative:

- no new package versions
- no broad folder moves
- no existing endpoint family rewrites
- no speculative dashboard DTOs
- no changes to existing operational analytics endpoints

## Added API Surface

The new endpoint group is mounted below the existing `/api` route group:

```text
/api/sql-operational/projects
/api/sql-operational/runs
/api/sql-operational/runs/{runId}
/api/sql-operational/runs/{runId}/manifest-rows
/api/sql-operational/runs/{runId}/work-items
/api/sql-operational/work-items/{workItemId}/complete
/api/sql-operational/work-items/{workItemId}/fail
/api/sql-operational/runs/{runId}/checkpoints
/api/sql-operational/runs/{runId}/asset-mappings
```

## Files


## Installer Behavior

The installer:

1. Copies the endpoint extension file.
2. Adds an Admin API project reference to `src/Core/Migration.Infrastructure.Sql`.
3. Registers the SQL operational store in `Program.cs`.
4. Maps the endpoint group in `AdminApiEndpointStartupExtensions.cs`.

## Validation

```powershell
./patches/P4.2-Validate-SqlOperationalAdminApiFacade.ps1

dotnet restore
dotnet build
```

## Notes

This is not the final project/run management API. It is the first functional SQL-backed facade so the Admin API can write to and read from the durable operational backbone created in P4.1.
