# P7.3 Hosted Worker Runtime Notes

This drop-in adds the hosted-service shell that turns the P7.2 SQL queue runtime into a continuously running worker process.

It intentionally does not mutate existing project files, package files, or appsettings files. Add the files, validate them, then wire the extension method from the actual cloud/worker host once you choose the final host project.

## Added files

- `src/Migration.Infrastructure/Runtime/Hosted/SqlOperationalWorkerOptions.cs`
- `src/Migration.Infrastructure/Runtime/Hosted/SqlOperationalWorkerHostedService.cs`
- `src/Migration.Infrastructure/Runtime/Hosted/SqlOperationalWorkerServiceCollectionExtensions.cs`
- `src/Migration.Infrastructure/Runtime/Hosted/SqlOperationalWorkerSampleExecutor.cs`

## Runtime behavior

When enabled, the hosted service:

1. reads `MigrationRuntime:SqlOperationalWorker` options from configuration
2. builds `SqlOperationalQueueRuntimeOptions`
3. starts either continuous processing or run-until-idle mode
4. claims work through the SQL queue runtime from P7.2
5. executes each claimed item through the configured executor delegate
6. relies on P7.2 to complete, fail, retry, or dead-letter work items
7. logs lifecycle and aggregate counters

## Why the executor is a delegate

The real connector execution path may live in existing P6 orchestration/application services. This set avoids adding another contract layer. The host supplies a `SqlOperationalWorkItemExecutor` delegate that can call the real P6 executor when you wire the actual cloud host.

A sample executor is included only as a smoke-test helper. It completes work items without performing a migration and should not be used as the production connector executor.
