# P4.1 SQL Operational Backbone

This drop-in adds the first SQL-first durable operational backbone as a new core infrastructure project:

```text
src/Core/Migration.Infrastructure.Sql
```

It intentionally does **not** inject SQL files into `src/Core/Migration.Infrastructure`.

## What this set adds

- `Migration.Infrastructure.Sql` project
- SQL connection factory
- SQL operational-store options and validation
- SQL operational backbone store
- durable operational records for projects, runs, manifest rows, work items, failures, checkpoints, mappings
- SQL schema script at `database/sql/operational/001_create_operational_backbone.sql`
- central package management update helper

## Package governance

The project uses central package management only. There are no inline `Version` attributes in `PackageReference` entries.

The install script adds these package versions to `Directory.Packages.props` only if missing:

- Dapper
- Microsoft.Data.SqlClient
- Microsoft.Extensions.Configuration.Abstractions
- Microsoft.Extensions.DependencyInjection.Abstractions
- Microsoft.Extensions.Options
- Microsoft.Extensions.Options.ConfigurationExtensions

## Install

From repo root, after extracting this package:

```powershell
./patches/P4.1-Install-SqlOperationalBackbone.ps1 -WhatIf
./patches/P4.1-Install-SqlOperationalBackbone.ps1 -Apply

dotnet restore
dotnet build
```

## Configuration section

```json
{
  "Migration": {
    "OperationalStore": {
      "Sql": {
        "ConnectionString": "Server=(localdb)\\MSSQLLocalDB;Database=MigrationBaseSolution;Integrated Security=true;TrustServerCertificate=true;",
        "CommandTimeoutSeconds": 60,
        "WorkItemLeaseMinutes": 15
      }
    }
  }
}
```

## Registration

In a host/API composition root:

```csharp
using Migration.Infrastructure.Sql.Registration;

builder.Services.AddSqlOperationalStore(builder.Configuration);
```

Do not wire this into `Migration.Admin.Api` yet unless the project is green after adding the SQL project. P4.2 should do Admin API integration deliberately.
