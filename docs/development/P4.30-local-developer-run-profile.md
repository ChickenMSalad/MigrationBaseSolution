# P4.30 — Local Developer Run Profile

This set explains how to use the new `migration-admin-ui` locally with `Migration.Admin.Api`.

## Local URLs

Recommended local split:

```text
Migration.Admin.Api       https://localhost:55436
migration-admin-ui        http://localhost:5174
```

The old Vite site can continue using:

```text
http://localhost:5173
```

The new operator UI can use:

```text
http://localhost:5174
```

## Manual run

Terminal 1:

```powershell
dotnet run --project src/Core/Migration.Admin.Api/Migration.Admin.Api.csproj --urls "https://localhost:55436"
```

Terminal 2:

```powershell
cd apps/migration-admin-ui
copy .env.local.example .env.local
npm install
npm run dev -- --host localhost --port 5174
```

Open:

```text
http://localhost:5174
```

## One-command helper

From repo root:

```powershell
./scripts/dev/Start-MigrationAdminLocal.ps1
```

That opens separate PowerShell windows for the API and UI.

## Smoke check

With the Admin API running:

```powershell
./scripts/dev/Test-MigrationAdminLocalProfile.ps1 -AllowUntrustedCertificate
```

## Notes

`npm run build` proves the UI compiles. `npm run dev` is what starts the actual local UI.
