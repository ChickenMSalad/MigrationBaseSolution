# P5.1.3 Azure Hosting Profiles

This set defines the first cloud hosting profile baseline for P5.

## Intended topology

- Admin/control-plane API: HTTP surface for operators and UI.
- Operator UI: browser-facing operator surface; no direct SQL access.
- Queue worker: long-running migration execution worker.
- Scheduler/dispatcher: controlled producer of queue work from durable SQL run state.
- Manifest ingestion: import process that turns CSV/Excel/artifacts into SQL operational data.

## SQL-first rule

SQL Server remains the durable operational source of truth for manifests, run queues, failures, reruns, replay state, source/target mapping, and execution history. Blob/file artifacts are ingestion/export/supporting artifacts, not the operational run system.

## Program.cs

No Program.cs change is required by this set.

Later P5 sets may add opt-in host composition that reads these profiles after the runtime topology options are already bound.
