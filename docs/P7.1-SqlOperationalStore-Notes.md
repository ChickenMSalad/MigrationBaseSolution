# P7.1 SQL Operational Store Notes

## Why this set exists

P6 created the runtime surfaces. P7.1 gives those surfaces a durable SQL Server backbone before we wire workers and dispatch loops.

## Included runtime tables

- `migration.MigrationRuns`
- `migration.ManifestRows`
- `migration.WorkItems`
- `migration.WorkItemFailures`
- `migration.IdentifierMappings`
- `migration.ExecutionCheckpoints`
- `migration.WorkerHeartbeats`
- `migration.ReleaseEvidence`

## Included procedures

- `migration.usp_ClaimWorkItems`
- `migration.usp_CompleteWorkItem`
- `migration.usp_FailWorkItem`
- `migration.usp_RecoverExpiredLeases`
- `migration.usp_RecordWorkerHeartbeat`

## Next set

P7.2 should wire the SQL queue procedures into the P6 runtime dispatcher and hosted execution loop.

The next set should inspect the exact P6 interfaces and register only the adapters that match the existing contracts. Do not introduce a new contract layer unless the current P6 contracts cannot express durable queue behavior.
