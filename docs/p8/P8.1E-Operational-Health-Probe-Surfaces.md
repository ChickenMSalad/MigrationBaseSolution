# P8.1E Operational Health Probe Surfaces

P8.1E defines the cloud probe contract for the SQL operational worker without changing runtime code.

## Probe model

- Liveness: the worker process/container is running.
- Startup readiness: `SqlOperationalWorkerStartupProbe` resolves `IOperationalRuntimeReadinessService` and requires the SQL operational runtime readiness report to be ready.
- Runtime idle: `SQL operational queue idle. No runnable migration runs found.` is a healthy state.
- RunId override: `SqlOperationalQueueExecutor:RunId` must remain unset except for short-lived debug runs.

## Azure hosting guidance

For App Service, Container Apps, or Kubernetes-style hosting, use platform process/container health for liveness and the startup log/readiness validation for readiness. A future API-hosted health endpoint should reuse the existing `IOperationalRuntimeReadinessService` rather than duplicate SQL checks.

## Required local/cloud settings

Use `MIGRATION_` environment variables in cloud:

- `MIGRATION_ConnectionStrings__MigrationOperationalStore`
- `MIGRATION_SqlOperationalQueueExecutor__Enabled`
- `MIGRATION_SqlOperationalQueueExecutor__WorkerId`
- `MIGRATION_SqlOperationalQueueExecutor__BatchSize`
- `MIGRATION_SqlOperationalQueueExecutor__LeaseSeconds`
- `MIGRATION_SqlOperationalQueueExecutor__PollDelaySeconds`
- `MIGRATION_SqlOperationalQueueExecutor__RunUntilIdleAndStop`
- `MIGRATION_SqlOperationalMigrationJobExecutor__Enabled`

Do not configure `MIGRATION_SqlOperationalQueueExecutor__RunId` outside short-lived debug runs.
