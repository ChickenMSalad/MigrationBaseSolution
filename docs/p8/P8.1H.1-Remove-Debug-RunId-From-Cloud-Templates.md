# P8.1H.1 Remove Debug RunId From Cloud Templates

## Purpose

`SqlOperationalQueueExecutor:RunId` is a debug-only override. It must not be present in Azure Container Apps, App Service, or production environment templates.

The SQL operational worker now discovers runnable runs from SQL through `GetRunnableRunIdsAsync`. Production/cloud workers should run with:

```text
RunIdOverride=(null)
```

## Required production behavior

Cloud templates should include:

```text
MIGRATION_SqlOperationalQueueExecutor__Enabled
MIGRATION_SqlOperationalQueueExecutor__WorkerId
MIGRATION_SqlOperationalQueueExecutor__BatchSize
MIGRATION_SqlOperationalQueueExecutor__LeaseSeconds
MIGRATION_SqlOperationalQueueExecutor__PollDelaySeconds
MIGRATION_SqlOperationalQueueExecutor__RunUntilIdleAndStop
```

Cloud templates must not include:

```text
MIGRATION_SqlOperationalQueueExecutor__RunId
```

## Local/debug behavior

A developer may temporarily configure `SqlOperationalQueueExecutor:RunId` through user-secrets or local environment variables to debug one run. Remove it immediately after the targeted debug session.

## Validation

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P81H1CloudTemplateRunIdPolicy.ps1
```

The validator fails if active cloud deployment templates contain the debug-only RunId setting.
