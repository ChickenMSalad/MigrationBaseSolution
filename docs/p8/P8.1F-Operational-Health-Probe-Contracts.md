# P8.1F Operational Health Probe Contracts

This set adds repo-native operational health probe contracts and a SQL-backed probe service without turning the SQL operational worker into an HTTP host.

## Purpose

P8.1F establishes reusable liveness/readiness payloads that can be exposed by the existing Admin API, a future lightweight probe sidecar, Container Apps, App Service, or Kubernetes hosting surfaces.

## Probe semantics

- Liveness means the process is running and can answer a basic in-process health call.
- Readiness means SQL operational runtime readiness reports ready through `IOperationalRuntimeReadinessService`.
- An idle worker with no runnable runs is healthy.
- `SqlOperationalQueueExecutor:RunId` remains debug-only and must not be required for readiness.

## Added surfaces

- `IOperationalHealthProbeService`
- `OperationalHealthProbeResponse`
- `OperationalHealthProbeDependencyStatus`
- `SqlOperationalHealthProbeService`
- `AddSqlOperationalHealthProbes`

## Next step
