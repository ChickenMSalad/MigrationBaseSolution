# P8.3D.1 Inventory-First Runtime Activity Usage

This set intentionally avoids runtime mutation scripts.

P8.3D attempted to patch worker files by exact string replacement. That proved too brittle. P8.3D.1 changes the workflow:

1. Validate the ActivitySource helper foundation exists.
2. Generate an inventory of the exact current SQL queue worker, Service Bus executor, and Service Bus dispatcher activity surfaces.
3. Use the generated inventory to apply a small manual patch in the next set, or manually after review.

## Intended runtime targets

- `src/Workers/Migration.Workers.QueueExecutor/Services/SqlOperationalWorkItemWorker.cs`
- `src/Workers/Migration.Workers.ServiceBusExecutor/Runtime/SqlServiceBusExecutorWorker.cs`
- `src/Workers/Migration.Workers.ServiceBusDispatcher/Dispatching/SqlWorkItemDispatcher.cs`

## Do not run old P8.3D repair scripts

Do not run these scripts from the previous set:

- `Repair-P83DApplySqlQueueWorkerActivityUsage.ps1`
- `Repair-P83DApplyServiceBusExecutorActivityUsage.ps1`
- `Repair-P83DApplyServiceBusDispatchActivityUsage.ps1`

They used brittle exact source matching and should be considered retired.

## Desired implementation shape

The worker files should eventually use the P8.3C helper methods:

- `OperationalExecutionActivity.StartSqlQueueWorkItemExecution(...)`
- `OperationalExecutionActivity.StartServiceBusWorkItemExecution(...)`
- `OperationalExecutionActivity.StartServiceBusDispatch(...)`
- `OperationalExecutionActivity.SetExecutionDuration(...)`
- `OperationalExecutionActivity.SetExecutionResult(...)`

The next implementation step should be manual or generated from the exact line inventory, not broad text replacement.
