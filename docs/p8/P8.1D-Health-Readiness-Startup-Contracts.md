# P8.1D Health / Readiness / Startup Contracts

## Purpose

P8.1D documents and validates the cloud health/readiness contract for the SQL operational worker host without changing runtime code.

The current repo already has a SQL operational readiness service and a worker startup probe. This set treats those as the repo-native readiness boundary for P8 cloud hosting.

## Cloud hosting contract

### Liveness

The process is live when the worker host process starts and remains running.

For worker-style hosts without HTTP endpoints, the cloud platform liveness signal should be process health:

- container/app process is running
- host has not crashed due to background-service failure
- logs are being emitted

### Readiness

The SQL operational worker is ready only when `SqlOperationalWorkerStartupProbe` can resolve `IOperationalRuntimeReadinessService` and the readiness report returns `IsReady = true`.

That readiness path validates the SQL operational runtime tables and connection path that the worker requires.

### Startup validation

Startup validation should fail fast for blocking SQL/runtime problems. It should not require a pinned run id. The worker should be valid with `SqlOperationalQueueExecutor:RunId` absent.

### Cloud configuration expectations

Required cloud configuration:

- `ConnectionStrings:MigrationOperationalStore`
- `SqlOperationalQueueExecutor:Enabled`
- `SqlOperationalQueueExecutor:WorkerId`
- `SqlOperationalQueueExecutor:BatchSize`
- `SqlOperationalQueueExecutor:LeaseSeconds`
- `SqlOperationalQueueExecutor:PollDelaySeconds`
- `SqlOperationalQueueExecutor:RunUntilIdleAndStop=false`

Do not configure `SqlOperationalQueueExecutor:RunId` except for short-lived debug runs.

## Validation

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P81DHealthReadinessStartupContracts.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\Write-P81DHealthReadinessInventory.ps1
```

Then run the worker in Development and confirm:

```text
RunIdOverride=(null)
SQL operational worker host readiness check passed
SQL operational queue idle. No runnable migration runs found.
```
