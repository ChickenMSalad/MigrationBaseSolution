# P8.2B Service Bus Execution Binding Contract Alignment

This set prepares the distributed Service Bus executor for real execution binding by inventorying the exact repo-native contracts currently used by:

- `Migration.Workers.ServiceBusExecutor`
- `Migration.Workers.ServiceBusDispatcher`
- SQL operational work item queue contracts
- SQL operational queue executor services
- execution-history writer surfaces

The goal is to avoid guessing at constructor signatures or message/result records before replacing the placeholder Service Bus executor.

## Why this exists

P8.2A proved that the repo already has substantial Service Bus dispatcher/executor infrastructure. The next implementation should bind Service Bus execution to real operational work-item execution, but only after confirming exact current symbols.

## Expected next step

After this inventory is generated, P8.2C should perform the targeted runtime patch:

- replace or decorate `PlaceholderServiceBusWorkItemExecutor`
- hydrate the SQL operational work item from the Service Bus message
- execute the same runtime execution boundary used by the SQL queue executor
- preserve SQL completion/failure/dead-letter behavior
- preserve execution-history recording
- avoid new package references unless explicitly required
