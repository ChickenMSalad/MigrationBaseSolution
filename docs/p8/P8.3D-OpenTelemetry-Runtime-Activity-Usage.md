# P8.3D OpenTelemetry Runtime Activity Usage

This set wires the Activity helper foundation from P8.3C into runtime execution paths with guarded PowerShell repair scripts.

## Scope

- SQL queue work-item execution activity usage.
- Service Bus executor work-item execution activity usage.
- Service Bus dispatcher send activity usage.
- Validation and inventory generation.

## Rules

- Preserve existing `ILogger.BeginScope` correlation scopes.
- Do not change queue settlement semantics.
- Do not change SQL completion/failure semantics.
- Do not add package references.
- Do not add SQL schema changes.
- Do not create a parallel telemetry model.

## Activity model

Use `OperationalExecutionActivity` helpers from `Migration.Application.Operational.Telemetry`:

- `StartSqlQueueWorkItemExecution`
- `StartServiceBusWorkItemExecution`
- `StartServiceBusDispatch`
- `SetExecutionDuration`
- `SetExecutionResult`

## Expected validation

After applying repair scripts, runtime files should contain:

- `OperationalExecutionActivity.StartSqlQueueWorkItemExecution`
- `OperationalExecutionActivity.StartServiceBusWorkItemExecution`
- `OperationalExecutionActivity.StartServiceBusDispatch`
- `OperationalExecutionActivity.SetExecutionDuration`
- `OperationalExecutionActivity.SetExecutionResult`
