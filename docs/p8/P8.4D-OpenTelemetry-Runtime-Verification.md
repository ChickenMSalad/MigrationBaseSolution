# P8.4D OpenTelemetry Runtime Verification

This set verifies that the OpenTelemetry runtime registration introduced in P8.4C is present and ready for local/cloud smoke testing.

## Goal

Confirm the operational runtime is now observable without changing runtime behavior.

Verification targets:

- SQL operational worker has OpenTelemetry registered.
- Service Bus dispatcher has OpenTelemetry registered.
- Service Bus executor has OpenTelemetry registered.
- The operational ActivitySource is registered by the shared extension.
- Azure Monitor exporter configuration is optional and environment-driven.
- Local development can run with tracing enabled and Azure Monitor exporter disabled.

## Local smoke configuration

For a local no-exporter smoke run, configure only the runtime host you are testing first.

Example for `src/Hosts/Migration.Hosts.SqlOperationalWorker`:

```powershell
dotnet user-secrets set "OpenTelemetry:EnableTracing" "true" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "OpenTelemetry:EnableAzureMonitorExporter" "false" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "OpenTelemetry:TraceSamplingRatio" "1.0" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
```

For Azure Monitor export, set either:

```powershell
dotnet user-secrets set "OpenTelemetry:AzureMonitorConnectionString" "<connection-string>" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "OpenTelemetry:EnableAzureMonitorExporter" "true" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
```

or use:

```powershell
$env:APPLICATIONINSIGHTS_CONNECTION_STRING = "<connection-string>"
```

## Recommended smoke order

1. Run the SQL operational worker locally with Azure Monitor exporter disabled.
2. Confirm the host starts and no OpenTelemetry registration exception occurs.
3. Run a small operational work item smoke path.
4. Enable Azure Monitor exporter only after the local no-exporter path succeeds.
5. Repeat for Service Bus dispatcher and executor when ready.

## Do not do in this set

- Do not add more package references unless build requires it.
- Do not mutate worker execution logic.
- Do not change SQL schemas.
- Do not change queue settlement behavior.
