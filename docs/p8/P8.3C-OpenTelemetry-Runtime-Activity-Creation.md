# P8.3C OpenTelemetry Runtime Activity Creation

This set adds an additive Activity helper around the canonical `OperationalExecutionActivitySources` and `OperationalExecutionActivityTags` introduced in P8.3B.

## Intent

- Keep Activity creation centralized.
- Avoid package/project changes until exporter wiring is selected.
- Preserve existing `ILogger.BeginScope` correlation scopes.
- Prepare SQL queue, Service Bus dispatch, and Service Bus execution paths for OpenTelemetry/Azure Monitor exporters.

## Runtime wiring guidance

Future runtime patches should create activities around:

- SQL queue work item execution in `SqlOperationalWorkItemWorker.ExecuteClaimedItemAsync`.
- Service Bus work item execution in `SqlServiceBusExecutorWorker` after message/work item resolution.
- Service Bus dispatch in `SqlWorkItemDispatcher` around message send.

Activity creation should complement, not replace, existing logging scopes and execution-history writes.
