# P8.3A OpenTelemetry / Azure Monitor Baseline

Purpose: establish the repo-level observability baseline before introducing OpenTelemetry or Azure Monitor package/runtime wiring.

This set is intentionally documentation and validation only. It does not add package references, project references, SQL, or runtime mutations.

## Baseline decisions

- Keep operational execution telemetry correlated by `RunId`, `WorkItemId`, and `ServiceBusCorrelationId`.
- Preserve existing `ILogger.BeginScope` usage for SQL queue and Service Bus executor runtime paths.
- Treat Application Insights / Azure Monitor as an exporter concern, not as a replacement for existing SQL execution history.
- Do not duplicate execution history in telemetry. Telemetry should point back to durable SQL evidence.
- Prefer repo-native Azure observability abstractions where they already exist.

## Target runtime surfaces

- Admin API control plane
- SQL operational worker
- Service Bus dispatcher worker
- Service Bus executor worker
- Connector execution runtime
