# P4.3 — SQL Manifest Provider

This drop-in replaces the placeholder `SqlManifestProvider` under:

```text
src/Manifests/Migration.Manifest.Sql/SqlManifestProvider.cs
```

The provider reads rows from SQL Server using the existing `MigrationJobDefinition.ConnectionString`
and `MigrationJobDefinition.QueryText` fields, or equivalent `Settings` values.

It intentionally does not add a new project, new DTO family, broad folder movement, or a second manifest abstraction.

## Run

```powershell
./patches/P4.3-Install-SqlManifestProvider.ps1
./patches/P4.3-Install-SqlManifestProvider.ps1 -Apply
./patches/P4.3-Validate-SqlManifestProvider.ps1

dotnet restore
dotnet build
```

## Expected manifest query columns

Recommended but not required:

```text
RowId
SourceAssetId
SourcePath
```

All returned SQL columns are copied into `ManifestRow.Columns`.

## Notes

- No inline package versions are added.
- No package versions are changed.
- No solution/project relocation is performed.
- The existing `Migration.Manifest.Sql.csproj` already references `Microsoft.Data.SqlClient`.
