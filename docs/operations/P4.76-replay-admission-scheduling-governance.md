# P4.76 — Replay Admission Scheduling Governance

This set adds replay admission governance.

Materialized replay sessions now start in:

```text
admission-pending
```

New route:

```text
POST /api/operational/execution-replay/admission/evaluate
```

Behavior:

- evaluates admission-pending replay sessions
- admits sessions into `queued` when allowed
- defers sessions when outside the UTC window
- defers sessions when replay concurrency is at limit
- writes `ExecutionReplayAdmitted`
- writes `ExecutionReplayDeferred`

Configuration sample:

```text
config-samples/appsettings.ExecutionReplayAdmission.sample.json
```
