# P4.81 — Durable Replay Admission Decision History

This set persists replay admission decisions.

New SQL migration:

```text
database/sql/operational/012_create_execution_replay_admission_decisions.sql
```

New route:

```text
GET /api/operational/execution-replay/{executionSessionId}/admission/history?take=25
```

Every admitted/deferred decision now records:

- execution session id
- decision
- reason
- active replay count
- max concurrent replay limit
- allowed-window result
- created timestamp

This makes replay admission behavior auditable after automated or manual admission evaluation.
