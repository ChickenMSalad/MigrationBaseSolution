# P4.71 — Replay Governance Approval Controls

This set adds durable replay approvals and active replay conflict prevention.

New SQL migration:

```text
database/sql/operational/009_create_execution_replay_approvals.sql
```

New route:

```text
POST /api/operational/execution-replay/approve
```

Governance behavior:

- approval requires operator identity
- approval requires approval note
- approval expires
- active replay conflicts are blocked
- materialization requires an active approval
- materialization consumes the approval
- approval writes `ExecutionReplayApproved`
