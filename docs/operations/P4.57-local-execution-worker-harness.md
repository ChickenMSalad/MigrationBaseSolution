# P4.57 — Local Execution Worker Harness

This set adds a local/dev worker harness that exercises the execution work-item queue.

## Prerequisites

1. Create an execution session from the UI.
2. Seed an execution plan.
3. Expand the work queue.
4. Copy the execution session ID.

## Complete work items

```powershell
./scripts/workers/P4.57-Invoke-LocalExecutionWorker.ps1 `
  -ExecutionSessionId "00000000-0000-0000-0000-000000000000" `
  -AllowUntrustedCertificate `
  -MaxIterations 10 `
  -BatchSize 2
```

## Simulate failures

```powershell
./scripts/workers/P4.57-Invoke-LocalExecutionWorker.ps1 `
  -ExecutionSessionId "00000000-0000-0000-0000-000000000000" `
  -AllowUntrustedCertificate `
  -FailLeasedItems `
  -MaxIterations 1 `
  -BatchSize 1
```

## What it does

The worker harness calls:

```text
POST /api/operational/execution-work-items/lease
POST /api/operational/execution-work-items/complete
POST /api/operational/execution-work-items/fail
```

This is intentionally a local harness, not the final production worker.
