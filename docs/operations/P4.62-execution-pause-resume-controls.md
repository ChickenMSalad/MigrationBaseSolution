# P4.62 — Execution Pause Resume Controls

This set adds session-level pause/resume controls and guards work-item leasing.

New routes:

```text
POST /api/operational/execution-control/pause
POST /api/operational/execution-control/resume
```

Paused, cancelled, completed, and failed sessions will not lease new work items.

Pause/resume writes correlated operational events:

- `ExecutionSessionPaused`
- `ExecutionSessionResumed`
- `ExecutionWorkItemLeaseSkipped`

This gives operators a safe way to stop new worker activity without deleting or mutating queued work.
