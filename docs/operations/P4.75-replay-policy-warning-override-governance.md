# P4.75 — Replay Policy Warning Override Governance

This set adds explicit replay policy warning overrides.

New SQL migration:

```text
database/sql/operational/011_create_execution_replay_policy_overrides.sql
```

New routes:

```text
POST /api/operational/execution-replay/policy/override
GET  /api/operational/execution-replay/{sourceExecutionSessionId}/policy/overrides?take=25
```

Rules:

- `allow` policy decisions do not need overrides.
- `warn` policy decisions can be overridden with operator identity and reason.
- `block` policy decisions cannot be overridden.
- Materialization requires an active override when policy decision is `warn`.
- Overrides expire and are consumed on successful materialization.
