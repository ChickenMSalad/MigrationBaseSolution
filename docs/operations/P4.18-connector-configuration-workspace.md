# P4.18 Connector Configuration Workspace

P4.18 adds the first operator-facing connector configuration workspace.

## Scope

This set adds:

- Admin API connector configuration summary endpoint
- Admin API connector catalog endpoint
- Admin API connector configuration validation endpoint
- React operator workspace for source/target connector configuration
- UI API client and TypeScript models

## Intent

This is not yet the credential persistence layer. It intentionally avoids storing secrets and avoids adding connector-specific runtime behavior.

The goal is to give operators a workspace for preparing connector configuration before launching a SQL-backed migration run.

## Added API Routes

```text
GET  /api/operational/connectors/configuration/summary
GET  /api/operational/connectors/configuration/catalog
POST /api/operational/connectors/configuration/validate
```

## Future Follow-Up

The next set should add SQL-backed connector registry persistence and credential-reference handling through Key Vault or an equivalent secret boundary.
