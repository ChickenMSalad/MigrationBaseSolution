# P4.67 — Execution Replay Readiness Analysis

This set adds deterministic replay-readiness analysis for execution sessions.

New route:

```text
GET /api/operational/execution-replay/{executionSessionId}/analysis
```

The analysis evaluates:

- session status
- plan completeness
- duplicate plan order values
- missing connector labels
- work-item state
- active leases
- failed/dead-lettered work
- critical operational events
- stale worker telemetry

The UI gets an **Analyze replay** action for selected execution sessions.

This does not replay anything yet. It creates the decision artifact needed before replay/resume/rerun workflows.
