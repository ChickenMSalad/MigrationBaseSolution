# P4.29 — Local SQL Runtime Smoke Workflow

This set adds a local smoke workflow for the SQL-backed operational runtime.

## Purpose

The goal is not to add another feature area. The goal is to make the current P4 runtime easier to verify after each drop-in.

The smoke workflow checks:

- SQL operational schema file presence
- required SQL backbone objects in the schema script
- Admin API composed command-center endpoint
- SLA/SLO endpoint
- notification endpoint
- capacity endpoint
- cost endpoint
- audit endpoint

## Commands

From repo root:

```powershell
./scripts/smoke/P4.29-Test-OperationalSqlSchemaFiles.ps1
dotnet restore
dotnet build
```

With the Admin API running:

```powershell
./scripts/smoke/P4.29-Invoke-LocalSqlRuntimeSmoke.ps1 -BaseUrl "https://localhost:55436" -AllowUntrustedCertificate
```

## Notes

This workflow intentionally does not create a database yet. Database provisioning should be handled by a later migration/deployment set once the SQL schema and deployment topology are stabilized.
