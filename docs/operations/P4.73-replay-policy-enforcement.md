# P4.73 — Replay Policy Enforcement

This set adds replay policy evaluation and gates materialization.

New route:

```text
GET /api/operational/execution-replay/{sourceExecutionSessionId}/policy?scope=failed-only
```

Policy decision values:

- `allow`
- `warn`
- `block`

Policy checks include:

- replay depth
- active replay conflict
- empty replay manifest
- replay volume
- dead-lettered percentage
- full-scope replay warning

Materialization now runs the policy preflight and blocks when decision is `block`.
