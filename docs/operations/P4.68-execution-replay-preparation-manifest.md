# P4.68 — Execution Replay Preparation Manifest

This set adds replay manifest preparation.

New route:

```text
POST /api/operational/execution-replay/prepare
```

Request:

```json
{
  "executionSessionId": "00000000-0000-0000-0000-000000000000",
  "scope": "failed-only",
  "reason": "operator recovery test"
}
```

Supported scopes:

- `failed-only`
- `dead-letter-only`
- `incomplete-only`
- `all`

The result is a replay-preparation artifact, not a new execution session.

This is intentionally approval-gated groundwork for future replay execution.
