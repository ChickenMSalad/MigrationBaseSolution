# P4.74 — Durable Replay Policy Evaluation Snapshots

This set persists replay policy evaluations for auditability.

New SQL migration:

```text
database/sql/operational/010_create_execution_replay_policy_evaluations.sql
```

New route:

```text
GET /api/operational/execution-replay/{sourceExecutionSessionId}/policy/history?take=25
```

Every policy evaluation is now stored with:

- source execution session id
- replay scope
- decision
- score
- metrics JSON
- violations JSON
- created timestamp

This makes replay governance decisions auditable after approval/materialization.
