# P10.2G - Admin Web Runtime Dashboard Route Wiring

## Purpose

P10.2D added the canonical Admin Web runtime dashboard client under `src/Admin/Migration.Admin.Web`. P10.2E/P10.2F established that `apps/migration-admin-ui` is feature-source only and that `src/Admin/Migration.Admin.Web` is the canonical deployable Admin UI.

This set wires the canonical Runtime Dashboard page into the canonical Admin Web route/nav surface without moving or mutating the feature-source app under `apps/migration-admin-ui`.

## Scope

Canonical target:

- `src/Admin/Migration.Admin.Web/src/App.tsx`
- `src/Admin/Migration.Admin.Web/src/components/Layout.tsx`

Expected result:

- `RuntimeDashboard` is imported from `./pages/RuntimeDashboard`.
- `/runtime-dashboard` is routed to `<RuntimeDashboard />`.
- The left navigation exposes `Runtime Dashboard`.

## Important guardrails

- Do not add new Admin UI feature work under `apps/migration-admin-ui`.
- Do not merge the entire feature-source app in one pass.
- Keep P10.2D as the last completed dashboard-client feature set before this route/nav wiring.
- Continue feature consolidation one bounded family at a time.
