# P10.2AG — Admin Web Notification Routing Feature Move

## Purpose

Continue P10.2 Admin UI consolidation by moving one governance feature family from flat Admin Web folders into the canonical feature structure.

## Scope

Moves these canonical Admin Web files:

- `src/Admin/Migration.Admin.Web/src/pages/NotificationRouting.tsx`
- `src/Admin/Migration.Admin.Web/src/api/notificationRoutingApi.ts`
- `src/Admin/Migration.Admin.Web/src/types/notificationRouting.ts`

Into:

- `src/Admin/Migration.Admin.Web/src/features/governance/notificationRouting/pages/NotificationRouting.tsx`
- `src/Admin/Migration.Admin.Web/src/features/governance/notificationRouting/api/notificationRoutingApi.ts`
- `src/Admin/Migration.Admin.Web/src/features/governance/notificationRouting/types/notificationRouting.ts`

Also updates:

- feature-local imports inside `NotificationRouting.tsx`
- the Admin Web `App.tsx` import for `NotificationRouting`
- the API client import path to continue using the existing canonical `api/core/adminApiClient`

## Non-goals

- No `/apps/migration-admin-ui` changes.
- No broad routing rewrite.
- No package changes.
- No build configuration changes.
