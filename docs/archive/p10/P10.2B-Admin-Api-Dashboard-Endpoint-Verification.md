# P10.2B — Admin API Dashboard Endpoint Verification

## Purpose

P10.2B intentionally does **not** patch `Migration.Admin.Api/Program.cs`.

The current repository already maps the runtime dashboard endpoints through:

- `using Migration.Admin.Api.Endpoints.Operational.Dashboard;`
- `app.MapSqlOperationalRuntimeDashboardEndpoints();`

The goal of this set is to verify the current Admin API route wiring, generate safe cloud verification commands, and prevent another redundant route-registration patch.

## Verified runtime dashboard endpoints

The Admin API dashboard endpoint extension exposes:

- `GET /api/runtime/dashboard/summary`
- `GET /api/runtime/dashboard/runs`

The endpoint implementation reads from the canonical P7.9 runtime SQL objects:

- `migration.Runs`
- `migration.WorkItems`

## Expected cloud settings

The Admin API must have at least one of the following connection settings:

- `ConnectionStrings__MigrationOperationalStore`
- `SqlOperationalRuntimeReadiness__ConnectionString`

`ConnectionStrings__MigrationOperationalStore` is preferred.

## Suggested flow

```powershell
.\tools\validate-p10.2b-admin-api-dashboard-endpoint-verification.ps1

.\tools\runtime\Test-P102AdminApiDashboardEndpointContract.ps1

.\tools\runtime\New-P102AdminApiDashboardCloudVerificationCommands.ps1 `
  -ResourceGroup $resourceGroup `
  -AdminApiAppName $adminApiApp `
  -BaseUrl "https://$adminApiApp.azurewebsites.net" `
  -OutputPath .\artifacts\p10\p10.2b-admin-api-dashboard-verify.ps1
```

Review the generated script before running it.

## Exit criteria

- Admin API project builds.
- Admin API App Service starts.
- `/api/runtime/dashboard/summary` returns HTTP 200.
- `/api/runtime/dashboard/runs` returns HTTP 200.
- The endpoints read the cleaned runtime SQL schema.
