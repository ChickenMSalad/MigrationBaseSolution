# P10.3H - Admin Web Manual UX Acceptance Checklist

Purpose: provide a focused manual click-through checklist after the Admin Web runtime gates are green.

## Preconditions

- Admin API is running locally.
- Admin Web is running locally.
- P10.3A through P10.3G have passed and been committed.
- Missing historical builder-feature parity remains deferred and should not block this site-up acceptance pass.

## Acceptance routes

| Area | Route | Expected result |
| --- | --- | --- |
| Home | `/` | App shell renders without a white screen. |
| Runtime Dashboard | `/runtime-dashboard` | Runtime dashboard page loads and handles empty data. |
| Runtime Detail | `/runtime-dashboard/:runId` | Detail route renders without breaking the shell. |
| Execution Sessions | `/execution-sessions` | Execution sessions page loads. |
| Failure Retry | `/failure-retry` | Failure retry page loads. |
| Operational Events | `/operations/operational-events` | Operational events page loads. |
| Connector Configuration | `/connector-configuration` | Connector configuration page loads. |
| Credential Vault | `/credential-vault` | Credential vault page loads. |
| Manifest Builder | `/manifest-builder` | Deferred page shell renders if present; feature parity remains tracked separately. |
| Taxonomy Builder | `/taxonomy-builder` | Deferred page shell renders if present; feature parity remains tracked separately. |
| Mapping Builder | `/mapping-builder` | Deferred page shell renders if present; feature parity remains tracked separately. |

## Manual checks

- Navigate every visible left-nav item once.
- Confirm the app shell/header/nav remains visible after each navigation.
- Confirm loading and empty states are readable.
- Confirm no obvious unhandled error block appears during navigation.
- Confirm core API-backed pages show data, empty state, or a controlled error message rather than a crash.
- Do not treat deferred historical builder parity as part of this gate.

## Evidence commands

Run the P10.3H checklist evidence runner after Admin Web and Admin API are already running:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\p10\P10.3H\Run-P10.3H-AdminWebManualUxAcceptanceChecklist.ps1 -AdminWebBaseUrl "http://localhost:5173" -AdminApiBaseUrl "https://localhost:55436"
```

The runner writes artifacts under `artifacts\p10\P10.3H`.
