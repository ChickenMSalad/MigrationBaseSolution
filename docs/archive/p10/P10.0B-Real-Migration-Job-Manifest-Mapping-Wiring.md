# P10.0B - Real Migration Job, Manifest, and Mapping Wiring

## Purpose

P10.0B turns the P10.0A selected real local-storage migration candidate into a concrete artifact set that can be inspected, seeded, and executed through the now-stabilized cloud runtime.

This set intentionally avoids changing runtime code. It adds only:

- a real job definition sample
- a real mapping profile sample
- a SQL seed script for the runtime `migration.Runs` / `migration.WorkItems` path
- a bounded validator for this set's files

## Contract

The selected candidate remains local-storage based so the first real execution validation does not reintroduce external API credential risk.

Canonical runtime assumptions:

- `RunId` remains `uniqueidentifier` / `Guid`.
- `WorkItemId` is owned by `migration.WorkItems` as `bigint identity`.
- `migration.WorkItems.RunId` points to `migration.Runs` after P7.9A.
- Runtime queue payload is a `MigrationJobDefinition` JSON payload.

## Files

- `profiles/jobs/p10-localstorage-real-migration.job.json`
- `profiles/mappings/p10-localstorage-real-migration.mapping.json`
- `database/sql/p10/001_seed_p10_localstorage_real_migration_work_item.sql`
- `config-samples/p10-localstorage-real-migration.sample.json`
- `tools/runtime/Test-P100RealMigrationArtifacts.ps1`

## Why this is not the final execution set

P10.0B only wires and validates artifacts. P10.0C should execute the seeded work item through dispatcher/executor and verify SQL state.
