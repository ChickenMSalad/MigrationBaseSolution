# P10.2Q Admin Web Notification Routing Client

## Purpose

Migrate the notification routing operator surface from the feature-source app into the canonical Admin Web application.

- Canonical deployable UI: `src/Admin/Migration.Admin.Web`
- Feature-source reference: `apps/migration-admin-ui`

This set is intentionally additive. It does not wire a route into `App.tsx` or navigation because recent route patchers have been brittle against the current one-line React source shape. Route wiring should be performed in a separate bounded route set or manually after confirming the committed `App.tsx` and `Layout.tsx` shapes.

## Added canonical Admin Web assets

- `src/Admin/Migration.Admin.Web/src/types/notificationRouting.ts`
- `src/Admin/Migration.Admin.Web/src/api/notificationRoutingApi.ts`
- `src/Admin/Migration.Admin.Web/src/pages/NotificationRouting.tsx`

## Expected API endpoints

The client targets the existing operational notification endpoints used by the feature-source workspace:

- `/api/operational/notifications/summary`
- `/api/operational/notifications/routes`
- `/api/operational/notifications/alert-preview`

## Validation

Run:

```powershell
.\tools\validate-p10.2q-admin-web-notification-routing-client.ps1
```

Then rebuild Admin Web from the repository root using the normal project workflow.
