# P10.2H - Admin Web Runtime Run Detail Client

## Purpose

P10.2H continues the Admin UI consolidation work after P10.2D/P10.2G by adding the canonical Admin Web run-detail client for the SQL operational runtime dashboard.

Canonical Admin UI remains:

`src/Admin/Migration.Admin.Web`

Feature-source/reference UI remains:

`apps/migration-admin-ui`

No new Admin UI work should target `apps/migration-admin-ui` as a deployable app.

## Scope

This set adds a Runtime Run Detail page to the canonical Admin Web and a bounded route-apply script that wires the page into the existing Vite/React router only when the expected P10.2G runtime dashboard route is present.

## Route

Expected route:

`/runtime-dashboard/:runId`

Expected API endpoint consumed by the page:

`/api/runtime/dashboard/runs/{runId}`

## Guardrails

- Does not mutate `apps/migration-admin-ui`.
- Does not add a second deployable UI.
- Does not touch Admin API backend code.
- Fails closed if `src/Admin/Migration.Admin.Web/src/App.tsx` does not contain a recognizable runtime dashboard route.
