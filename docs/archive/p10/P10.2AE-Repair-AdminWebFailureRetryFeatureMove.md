# P10.2AE Repair — Admin Web Failure Retry Feature Move

## Why this exists

The original P10.2AE script expected `src/api/failureRetry.ts`, but the repo uses `src/api/failureRetryApi.ts`.

The original apply script also moved the page before discovering the missing API source. This repair is intentionally idempotent around both clean and partially-applied states.

## Canonical target

```text
src/Admin/Migration.Admin.Web/src/features/operations/failureRetry/
  api/failureRetryApi.ts
  pages/FailureRetry.tsx
  types/failureRetry.ts
```

`src/Admin/Migration.Admin.Web/src/App.tsx` should import:

```ts
import { FailureRetry } from "./features/operations/failureRetry/pages/FailureRetry";
```

## Guardrails

- PowerShell 5.1 compatible
- `Set-StrictMode -Version 2.0` safe
- Preflights all required source-or-destination files before moving
- Uses `[pscustomobject]` for structured move data
- Does not scan its own validator text
