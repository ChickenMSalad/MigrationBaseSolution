# P10.0C - Real Migration Runtime Execution

## Purpose

P10.0C moves the selected P10 local-storage migration path from artifact wiring into the stabilized SQL/Service Bus runtime execution lane.

This set does not introduce a new migration engine. It uses the existing runtime contract that was stabilized during P7.9/P7.10:

- `migration.Runs` is the canonical run parent.
- `migration.WorkItems` is the canonical SQL operational queue table.
- `WorkItemId` is a `bigint` identity.
- `RunId` remains a `uniqueidentifier`.
- The dispatcher claims queued SQL work and publishes to Service Bus.
- The executor consumes Service Bus messages and executes `MigrationJobDefinition` payloads.

## Selected execution target

The selected P10 path remains the repo-native local-storage real migration candidate introduced in P10.0A/P10.0B:

- Job definition: `profiles/jobs/p10-localstorage-real-migration.job.json`
- Mapping profile: `profiles/mappings/p10-localstorage-real-migration.mapping.json`
- Manifest: `profiles/manifests/localstorage-platform-smoke.csv` or the P10.0A candidate manifest
- Source samples: `samples/localstorage/source`

## What this set adds

This set adds a bounded execution harness and state validator:

- `Invoke-P100RealMigrationRuntimeExecution.ps1` creates a canonical SQL run and queues one `MigrationJobDefinition` work item.
- `Test-P100RealMigrationRuntimeState.ps1` captures the SQL state for the P10 run/work item after dispatcher/executor processing.
- `002_p10_real_migration_runtime_state_validator.sql` verifies the database has the canonical objects and reports the latest P10 work state.

## Run sequence

From repo root:

```powershell
$runId = [guid]::NewGuid()
$runId
```

```powershell
.\tools\runtime\Invoke-P100RealMigrationRuntimeExecution.ps1 `
  -SqlServer $sqlServer `
  -Database $db `
  -SqlAdmin $sqlAdmin `
  -SqlPasswordPlain $sqlPasswordPlain `
  -RunId $runId
```

Restart dispatcher after enqueue:

```powershell
az webapp restart `
  --resource-group $resourceGroup `
  --name $dispatcherApp
```

After executor has had time to process:

```powershell
.\tools\runtime\Test-P100RealMigrationRuntimeState.ps1 `
  -SqlServer $sqlServer `
  -Database $db `
  -SqlAdmin $sqlAdmin `
  -SqlPasswordPlain $sqlPasswordPlain `
  -RunId $runId
```

## Exit criteria

The target state for P10.0C is:

```text
Queued -> Dispatched -> Completed
```

A controlled `Failed` state is acceptable only when the failure proves a real connector/artifact issue that must become the next P10.0D fix. It should not fail because of missing SQL parent rows, stale app settings, missing provider registration, or loose smoke artifacts.

## Non-goals

This set does not deploy workers, mutate app settings, or clean cloud infrastructure. Those paths were already stabilized and should remain separate.
