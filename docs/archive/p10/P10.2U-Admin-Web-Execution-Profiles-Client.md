# P10.2U — Admin Web Execution Profiles Client

## Purpose

Migrate the execution profile operator view from the feature-source UI into the canonical Admin Web application.

- Canonical deployable UI: `src/Admin/Migration.Admin.Web`
- Feature-source/reference UI: `apps/migration-admin-ui`

This set adds the client-side execution profile page, API wrapper, and TypeScript contracts to the canonical Admin Web. It does not wire routes automatically because the canonical React route source has been minified/one-line in recent commits and route patching has proven fragile.

## Added canonical files

- `src/Admin/Migration.Admin.Web/src/types/executionProfiles.ts`
- `src/Admin/Migration.Admin.Web/src/api/executionProfilesApi.ts`
- `src/Admin/Migration.Admin.Web/src/pages/ExecutionProfiles.tsx`

## Source feature family

The source feature family is `apps/migration-admin-ui/src/features/executionProfiles`.
