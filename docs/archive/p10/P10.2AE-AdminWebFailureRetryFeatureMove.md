# P10.2AE - Admin Web Failure Retry Feature Move

## Intent

Continue P10.2 Admin UI consolidation by moving one operations feature family at a time into the canonical feature folder structure.

## Scope

Canonical UI only:

- `src/Admin/Migration.Admin.Web/src/pages/FailureRetry.tsx`
- `src/Admin/Migration.Admin.Web/src/api/failureRetry.ts`
- `src/Admin/Migration.Admin.Web/src/types/failureRetry.ts`
- `src/Admin/Migration.Admin.Web/src/App.tsx`

Target feature structure:

- `src/Admin/Migration.Admin.Web/src/features/operations/failureRetry/pages/FailureRetry.tsx`
- `src/Admin/Migration.Admin.Web/src/features/operations/failureRetry/api/failureRetry.ts`
- `src/Admin/Migration.Admin.Web/src/features/operations/failureRetry/types/failureRetry.ts`

## Non-goals

- Do not touch `/apps/migration-admin-ui`.
- Do not create a second Admin UI.
- Do not change routes beyond import path repair.
- Do not change backend API contracts.
- Do not mass move unrelated pages.

## Validation

The test script checks:

- source files moved out of flat locations
- target files exist
- `App.tsx` imports `FailureRetry` from the feature folder
- moved feature files do not retain stale flat `../api/failureRetry`, `../types/failureRetry`, or `../lib/` imports
