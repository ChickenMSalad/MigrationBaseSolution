# P10.2CA - Admin Web Navigation Deduplication

## Purpose

After Admin Web consolidation compiled successfully, the remaining cleanup moves from import/build repair to usability hygiene.

This set normalizes `Layout.tsx` navigation so the canonical Admin Web shell does not show duplicate navigation links and does not accidentally omit migrated pages such as Operational Events when a matching route exists.

## Scope

- Reads local `App.tsx` route paths.
- Replaces only the `const nav = [...]` block in `Layout.tsx`.
- Keeps navigation entries route-backed by the local app.
- Writes a report to `docs/P10/P10.2CA-AdminWebNavigationDeduplication.Report.md`.

## Out of scope

- No route additions.
- No App.tsx changes.
- No feature moves.
- No package changes.
