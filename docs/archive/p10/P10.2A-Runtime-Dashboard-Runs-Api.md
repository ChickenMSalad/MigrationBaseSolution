# P10.2A - Runtime Dashboard Runs API

## Purpose

Expose a small Admin API read-only dashboard surface for the stabilized SQL runtime so the cloud site can show real run/work-item state before adding richer UI controls.

## Repo-aligned locations

- Admin API host: `src/Core/Migration.Admin.Api`
- Existing endpoint area: `src/Core/Migration.Admin.Api/Endpoints/Operational`
- New endpoint extension: `src/Core/Migration.Admin.Api/Endpoints/Operational/Dashboard/RuntimeDashboardEndpointExtensions.cs`

The repository README identifies the Control Plane as an ASP.NET Minimal API plus React UI, and the Admin API directory already has endpoint folders for Operational, Runs, Queue, System, and related areas.

## Endpoint surface

Adds read-only endpoints:

- `GET /api/runtime/dashboard/summary`
- `GET /api/runtime/dashboard/runs?take=50`

Both endpoints read from canonical runtime tables:

- `migration.Runs`
- `migration.WorkItems`

No write/mutation behavior is introduced in this set.

## Apply step

After copying the drop-in files, run:

```powershell
.\tools\runtime\Apply-P102RuntimeDashboardEndpoints.ps1
```

Then build the Admin API.

## Exit criteria

- Validator passes.
- Admin API builds.
- Admin API starts.
- `/api/runtime/dashboard/summary` returns JSON from the cloud runtime database.
- `/api/runtime/dashboard/runs` returns recent runtime runs.
