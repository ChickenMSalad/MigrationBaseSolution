# P10.2C - Runtime Dashboard Run Detail API

## Purpose

Add the next runtime-dashboard read slice after P10.2A/P10.2B: per-run detail, per-run work items, and per-run failed work-item detail.

This set is based on the current `main` repo shape:

- `src/Core/Migration.Admin.Api/Program.cs` already imports `Migration.Admin.Api.Endpoints.Operational.Dashboard`.
- `Program.cs` already maps `app.MapSqlOperationalRuntimeDashboardEndpoints();`.
- `RuntimeDashboardEndpointExtensions.cs` already exposes `/api/runtime/dashboard/summary` and `/api/runtime/dashboard/runs`.

P10.2C adds a separate detail extension and an apply script that inserts one additional app-level mapping directly after the existing dashboard mapping.

## New endpoints

```text
GET /api/runtime/dashboard/runs/{runId}
GET /api/runtime/dashboard/runs/{runId}/work-items?take=100
GET /api/runtime/dashboard/runs/{runId}/failures?take=100
```

## Data sources

The endpoints only read the canonical runtime tables proven during P7.9/P7.10 cleanup:

```text
migration.Runs
migration.WorkItems
```

They do not query legacy GUID-era tables.

## Apply

```powershell
.\tools\runtime\Apply-P102RuntimeDashboardRunDetailEndpoints.ps1
.\tools\validate-p10.2c-runtime-dashboard-run-detail-api.ps1
```

Then rebuild `Migration.Admin.Api`.
