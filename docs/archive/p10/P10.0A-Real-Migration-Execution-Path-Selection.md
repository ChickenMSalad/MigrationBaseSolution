# P10.0A - Real Migration Execution Path Selection

## Goal

Resume real platform work after runtime stabilization by selecting the smallest **real** migration path that can run through the stabilized cloud dispatcher/executor pipeline.

This set intentionally does **not** add another NoOp/Smoke-only runtime path. It selects a concrete connector path already present in the repo and prepares the minimum fixture needed for follow-up execution.

## Selected path

Use the existing LocalStorage-to-LocalStorage path as the first real migration execution validation target:

```text
Csv manifest
  -> LocalStorage source connector
  -> LocalStorage target connector
  -> GenericMigrationJobRunner
  -> SQL operational work item lifecycle
```

## Repo evidence reviewed

The repo already contains:

```text
profiles/jobs/localstorage-platform-smoke.job.json
profiles/mappings/localstorage-platform-smoke.json
src/Manifests/Migration.Manifest.Csv/CsvManifestProvider.cs
src/Connectors/Sources/Migration.Connectors.Sources.LocalStorage/LocalStorageSourceConnector.cs
src/Connectors/Targets/Migration.Connectors.Targets.LocalStorage/LocalStorageTargetConnector.cs
```

The existing job definition points to:

```text
profiles/manifests/localstorage-platform-smoke.csv
samples/localstorage/source
samples/localstorage/target
```

The mapping profile already exists, but the manifest/source fixture was not present in the earlier review bundle. This set adds a deterministic fixture so the job has a real manifest and real local files.

## Why this path

This is the lowest-risk real migration because:

- it uses an existing real manifest provider: `Csv`
- it uses existing real connectors: `LocalStorage` source and target
- it avoids Bynder/AEM/SharePoint credentials while proving real binary movement semantics
- it exercises manifest loading, source resolution, target execution, and completion handling
- it can run locally first and then be packaged into cloud execution in P10.0B/P10.0C

## Exit criteria for P10.0A

P10.0A is complete when:

- the selected path is documented
- required job, mapping, manifest, and fixture files exist
- fixture manifest references real source files
- the validator passes

## Not in this set
