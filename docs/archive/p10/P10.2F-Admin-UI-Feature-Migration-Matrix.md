# P10.2F - Admin UI Feature Migration Matrix

## Purpose

P10.2F continues the Admin UI consolidation corrective track started in P10.2E.

The canonical Admin UI is:

```text
src/Admin/Migration.Admin.Web
```

The feature source that must be migrated, not extended in place, is:

```text
apps/migration-admin-ui
```

This set does not move source files. It creates a repeatable inventory/matrix generator so each feature family can be migrated deliberately into the canonical site without creating a third UI path or breaking the current working Admin Web build.

## Why this exists

The repo contains two substantial Admin UI surfaces. P10.2D added runtime dashboard client files to the canonical solution-integrated Admin Web. P10.2E documented that future Admin UI work must target the canonical site only. P10.2F turns that decision into an actionable migration matrix.

## Output

The inventory script writes:

```text
artifacts/admin-ui-consolidation/p10.2f-admin-ui-feature-migration-matrix.md
artifacts/admin-ui-consolidation/p10.2f-admin-ui-feature-migration-matrix.json
```

The matrix groups source UI files by feature family and marks the intended destination under `src/Admin/Migration.Admin.Web`.

## Rules

- Do not add new feature work under `apps/migration-admin-ui`.
- Do not delete `apps/migration-admin-ui` yet.
- Migrate feature families one at a time.
- Rebuild canonical Admin Web after each code-moving set.
- Keep `apps/migration-admin-ui` as a source/reference until the migrated feature family has equivalent or better coverage in `src/Admin/Migration.Admin.Web`.
