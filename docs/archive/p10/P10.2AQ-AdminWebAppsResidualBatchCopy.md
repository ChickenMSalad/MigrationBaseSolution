# P10.2AQ - Admin Web Apps Residual Batch Copy

## Goal

Batch-copy residual source assets from the non-deployable reference UI into the canonical Admin Web so future work can focus on wiring, import normalization, and build validation instead of repeated single-feature moves.

## Scope

Source root:

`apps/migration-admin-ui/src`

Canonical target root:

`src/Admin/Migration.Admin.Web/src`

Families copied when missing:

- `features`
- `components`
- `auth`
- `lib`

## Out of scope

- No route changes.
- No `App.tsx` mutation.
- No package changes.
- No delete/retire of `/apps`.
- No overwrite of canonical files.

## Why this set exists

Prior feature-by-feature movement was too slow and produced fragile path/import validators. This set moves the remaining reference source into canonical safely while preserving committed canonical work.
