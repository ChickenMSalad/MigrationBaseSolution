# P10.2S — Admin Web Command Center Client

## Purpose

Migrate the command-center client surface from the feature-source app into the canonical Admin Web app.

## Canonical UI rule

- Canonical deployable UI: `src/Admin/Migration.Admin.Web`
- Feature-source reference only: `apps/migration-admin-ui`

## Scope

This set adds a bounded Command Center client/page to the canonical Admin Web app. It does not wire routes automatically and does not make `apps/migration-admin-ui` deployable.

## Added canonical files

- `src/Admin/Migration.Admin.Web/src/types/commandCenter.ts`
- `src/Admin/Migration.Admin.Web/src/api/commandCenterApi.ts`
- `src/Admin/Migration.Admin.Web/src/pages/CommandCenter.tsx`

## Follow-up

Route/navigation wiring should be done only after inspecting the current committed `App.tsx` and `Layout.tsx` shapes. Prior route apply scripts proved too fragile against minified one-line React source.
