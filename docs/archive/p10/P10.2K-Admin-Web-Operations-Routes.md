# P10.2K — Admin Web Operations Routes

## Purpose

P10.2K wires the migrated operational pages into the canonical Admin Web application under `src/Admin/Migration.Admin.Web`.

This continues the consolidation rule established by P10.2E:

- Canonical Admin UI: `src/Admin/Migration.Admin.Web`
- Feature source only: `apps/migration-admin-ui`

## Scope

This set wires route/navigation entries for pages already migrated into the canonical Admin Web:

- `ExecutionSessions.tsx`
- `FailureRetry.tsx`

It does not copy additional feature code from `apps/migration-admin-ui`, and it does not make `/apps/migration-admin-ui` deployable.

## Expected routes

- `/execution-sessions`
- `/failure-retry`

## Apply

Run:

```powershell
.\tools\runtime\Apply-P102AdminWebOperationsRoutes.ps1
.\tools\validate-p10.2k-admin-web-operations-routes.ps1
```

Then rebuild the canonical Admin Web.
