# P10.2AB - Admin Web Runtime Dashboard Feature Move

## Scope

This set starts moving canonical Admin Web code from flat folders into the feature structure created during P10.2Z.

It intentionally moves only the runtime dashboard page family:

- `RuntimeDashboard.tsx`
- `RuntimeRunDetail.tsx`

## Canonical UI Rule

The deployable Admin UI remains:

`src/Admin/Migration.Admin.Web`

The reference-only UI remains:

`apps/migration-admin-ui`

No file under `/apps` is modified by this set.

## Why this set is small

Previous route patchers were fragile because the current `App.tsx` shape differed from expectations. This set does not rewrite routes. It only changes the two import paths needed after moving the two files.

## Validation

Run:

```powershell
.\tools\p10\P10.2AB\Test-P10.2AB-AdminWebRuntimeDashboardFeatureMove.ps1
```

Then build the canonical Admin Web normally.
