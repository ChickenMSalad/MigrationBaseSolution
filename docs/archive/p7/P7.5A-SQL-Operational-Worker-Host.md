# P7.5A SQL Operational Worker Host

This set replaces the bad P7.5 path assumption. The worker host belongs under `src/Hosts`, and infrastructure remains under `src/Core`.

## Adds

- `src/Hosts/Migration.Hosts.SqlOperationalWorker/Migration.Hosts.SqlOperationalWorker.csproj`
- `src/Hosts/Migration.Hosts.SqlOperationalWorker/Program.cs`
- `src/Hosts/Migration.Hosts.SqlOperationalWorker/appsettings.json`
- `src/Hosts/Migration.Hosts.SqlOperationalWorker/appsettings.Development.json`
- `scripts/Validate-P7SqlOperationalWorkerHost.ps1`

## Does not add

- `src/Migration.Infrastructure`
- `src/Migration.Worker`
- inline package versions
- SQL scripts
- mutation scripts

## Apply

Copy the files into the repository root.

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P7SqlOperationalWorkerHost.ps1
```

Add the host project:

```powershell
dotnet sln add .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
```

If the host project needs `Microsoft.Data.SqlClient`, add it centrally to `Directory.Packages.props` and then add the package reference without an inline Version.

```powershell
dotnet add .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj package Microsoft.Data.SqlClient
```

Then remove any inline `Version` attribute if the CLI adds one, and keep the version centralized only.

## Run

Apply SQL scripts in order first:

```text
database/sql/p7/001_operational_runtime_store.sql
database/sql/p7/002_operational_queue_procedures.sql
database/sql/p7/003_operational_queue_runtime_wiring.sql
```

Set `ConnectionStrings:MigrationOperationalStore`.

Run:

```powershell
dotnet run --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj --environment Development
```

## Operational behavior

The host starts the P7 SQL operational runtime composition, validates SQL readiness, then runs the hosted SQL operational worker registered by the prior P7 runtime sets. In Development, `RunUntilIdleAndStop` is true so local smoke runs can drain available work and exit.
