# P7.10F Runtime Handoff Index

## Purpose

This set consolidates the cleaned runtime deployment path into a single handoff index. It does not mutate code, SQL, app settings, or cloud resources.

## Canonical runtime path

Use the P7.9/P7.10 cleaned path only:

1. Apply and validate runtime SQL canonicalization.
2. Use canonical unprefixed runtime App Service settings.
3. Build and deploy dispatcher and executor from clean publish output.
4. Run the RuntimeSmoke/NoOp smoke path.
5. Capture deployment evidence.
6. Compare local/Azure runtime parity.

## Do not use

Do not use the old Csv smoke path for runtime readiness. Csv remains a legitimate migration manifest/provider for Csv jobs, but it is not the runtime smoke readiness path.

Do not use loose `smoke.json` as the runtime smoke mapping profile. Use the packaged RuntimeSmoke/NoOp smoke assets.

Do not rely on historical `.migration-control-plane` run/project snapshots as deployment truth.

## Required validation before handoff

Run these in order:

```powershell
.\tools\validate-p7.9j-runtime-cleanup-closeout.ps1
.\tools\validate-p7.10a-retire-csv-smoke-path.ps1
.\tools\validate-p7.10b-canonical-sql-baseline-handoff.ps1
.\tools\validate-p7.10c-local-azure-runtime-parity.ps1
.\tools\validate-p7.10d-ci-runtime-release-gate.ps1
.\tools\validate-p7.10e-stale-runtime-artifact-quarantine.ps1
.\tools\validate-p7.10f-runtime-handoff-index.ps1
```

## Evidence expected

The final evidence bundle should include:

- Runtime SQL validator output
- Canonical appsettings export
- Dispatcher and executor deployment timestamps
- RuntimeSmoke/NoOp smoke state output
- Queue state after smoke
- Worker logs showing RunStarted, ManifestLoaded, and RunCompleted

## Owner notes

This handoff index intentionally references existing validators instead of scanning the entire repository again. Earlier failures came from broad validators flagging legitimate historical artifacts and their own rule text.
