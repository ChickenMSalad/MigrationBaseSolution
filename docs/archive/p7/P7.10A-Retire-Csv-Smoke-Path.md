# P7.10A Retire Csv Smoke Path

## Purpose

The runtime smoke path is now `RuntimeSmoke` through the packaged NoOp smoke provider. The old ad hoc smoke path that used `manifestType = Csv` and loose `smoke.json` files must not be used for operational runtime smoke validation.

This does **not** retire Csv migration support. Csv may still be a valid real migration manifest/provider type in console samples, historical control-plane artifacts, and migration examples.

## Validation boundary

The P7.10A validator only checks active runtime smoke assets:

- `tools/runtime/*Smoke*.ps1`
- `config-samples/*smoke*.json`
- `database/sql/p7/*smoke*.sql`
- `profiles/jobs/*smoke*.json`

It intentionally excludes:

- `.migration-control-plane/**`
- `src/Core/Migration.Admin.Api/Runtime/**`
- `docs/**` except this P7.10A document presence validation
- console sample jobs such as Csv or WebDAM examples
- validator scripts themselves
- `bin`, `obj`, and `.git` path segments

## Expected result

Active runtime smoke should reference the NoOp smoke path:

- manifest type: `RuntimeSmoke`
- source type: `RuntimeSmoke`
- target type: `RuntimeSmoke`
- mapping profile: `runtime-smoke.mapping.json`

Active runtime smoke should not reference:

- manifest type: `Csv`
- loose mapping profile path: `smoke.json`
