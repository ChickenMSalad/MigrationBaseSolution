# P7.6A QueueExecutor SQL Operational Wiring Inventory

This set is intentionally a repo-native safety and discovery set before modifying runtime code.

It does not add compiled C# files and should not affect build output.

## Why this exists

P7.5 exposed an important correction: the repo already has real runtime surfaces under:

- `src/Core/Migration.Infrastructure.Sql`
- `src/Core/Migration.Orchestration`
- `src/Core/Migration.GenericRuntime`
- `src/Workers/Migration.Workers.QueueExecutor`

Future operational wiring must extend those surfaces instead of inventing new `Migration.Infrastructure.Runtime` namespaces or top-level `src/Migration.*` projects.

## What to run after drop-in

From the repo root:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P7QueueExecutorWiringPrerequisites.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\Write-P7QueueExecutorWiringInventory.ps1
```

The inventory script writes:

```text
docs/p7/P7.6A-QueueExecutor-Wiring-Inventory.generated.md
```

## What this validates

- bad fabricated folders are absent
- bad `Migration.Infrastructure.Runtime` references are absent
- P7 SQL scripts `001`, `002`, and `003` exist
- repo-native runtime backbone projects exist
- `Migration.Workers.QueueExecutor` exists
- project files do not use inline `PackageReference Version` attributes

## Expected next implementation set

The next implementation set should use the generated inventory to wire SQL operational queue execution into the existing queue executor path.

Expected target areas:

- SQL operational queue claim/complete/fail/retry calls
- existing QueueExecutor worker entry point
- existing GenericRuntime and Orchestration execution types
- existing connector registration and execution paths

## Do not reintroduce

Do not add:

```text
src/Migration.Infrastructure
src/Migration.Worker
Migration.Infrastructure.Runtime.*
```

Those were incorrect for this repo.
