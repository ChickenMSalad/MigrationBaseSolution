# P7.8A Runtime Contract Cleanup

## Purpose

This set freezes and validates the runtime contract that was proven during the cloud smoke debugging session. It is intentionally conservative: it adds documentation, app-settings samples, SQL validation, and PowerShell validation/export tools. It does not delete legacy SQL objects and does not mutate runtime worker code.

## Why this set exists

The live smoke exposed contract drift across SQL, code, and Azure App Service configuration:

- `migration.WorkItems` and `migration.ManifestRows` are the active SQL-backed runtime queue tables.
- `migration.MigrationWorkItems`, `migration.MigrationManifestRecords`, and related GUID-era objects still exist and are legacy compatibility objects.
- `WorkItemId` and `ManifestRowId` in the active runtime queue model are `bigint` identity values.
- `RunId` remains `uniqueidentifier` / `Guid`.
- Dispatcher and executor app settings had multiple duplicate naming families, which made it unclear which settings were actually bound.

## Canonical runtime contract

| Concept | Canonical location/type | Owner |
| --- | --- | --- |
| Run ID | `uniqueidentifier` / `Guid` | Application/runtime correlation |
| Work item ID | `migration.WorkItems.WorkItemId bigint identity` | SQL runtime queue |
| Manifest row ID | `migration.ManifestRows.ManifestRowId bigint identity` | SQL runtime manifest rows |
| Work item table | `migration.WorkItems` | Active runtime queue |
| Manifest rows table | `migration.ManifestRows` | Active runtime manifest rows |
| Service Bus message `WorkItemId` | `long` JSON number | Dispatcher payload |
| Service Bus message `RunId` | `Guid` JSON string | Dispatcher payload |
| Service Bus message `SequenceNumber` | `long` JSON number | Dispatcher payload |

## Validation commands

### Validate repo drop-in placement

```powershell
.\tools\validate-p7.8a-runtime-contract-cleanup.ps1
```

### Validate exported App Service settings JSON

```powershell
.\tools\runtime\Test-RuntimeAppSettings.ps1 `
  -DispatcherSettingsPath .\dispatcher-appsettings.json `
  -ExecutorSettingsPath .\executor-appsettings.json
```

### Validate Azure SQL contract

```powershell
sqlcmd `
  -S "$sqlServer.database.windows.net" `
  -d $db `
  -U $sqlAdmin `
  -P $sqlPasswordPlain `
  -i .\database\sql\p7\008_runtime_contract_validator.sql
```

Or through PowerShell:

```powershell
.\tools\runtime\Test-RuntimeSqlContract.ps1 `
  -SqlServer "$sqlServer.database.windows.net" `
  -DatabaseName $db `
  -SqlUser $sqlAdmin `
  -SqlPassword $sqlPasswordPlain
```

## What this set intentionally does not do

- It does not delete `migration.MigrationWorkItems` or other GUID-era compatibility tables.
- It does not change `src` runtime code.
- It does not normalize SQL deployment scripts yet.
- It does not produce a final handoff runbook.

Those are next sets after the contract validators are in place.

P7.8B should consolidate SQL deployment scripts into a handoffable baseline and should explicitly mark GUID-era scripts as legacy or compatibility-only.
