# P7.9D Runtime NoOp Smoke Harness

## Purpose

This set replaces the earlier ad hoc smoke payloads that used `Csv`, `SmokeManifest`, and loose `smoke.json` files with a canonical SQL operational smoke path that targets the repo's runtime smoke provider introduced in P7.9C.

The smoke harness is intentionally narrow:

- It seeds a run into both current run tables while the runtime FK drift is being retired.
- It enqueues a `migration.WorkItems` row with `WorkType = MigrationJobDefinition`.
- It uses the runtime smoke discriminator values:
  - `manifestType = RuntimeSmoke`
  - `sourceType = RuntimeSmoke`
  - `targetType = RuntimeSmoke`
- It references `runtime-smoke.mapping.json`, which should be packaged with the ServiceBusExecutor by P7.9C.

## Why this exists

The previous smoke flow proved the platform path but failed for the wrong reasons:

- fake `Csv` manifest provider assumptions
- missing `smoke.json`
- incomplete mapping profile shape
- run parent FK drift between `migration.Runs` and `migration.MigrationRuns`

This set keeps smoke validation focused on runtime plumbing, not connector/provider discovery guesses.

## Use

From repo root:

```powershell
$runId = [guid]::NewGuid()

.\tools\runtime\Invoke-RuntimeNoOpSmoke.ps1 `
  -SqlServer $sqlServer `
  -Database $db `
  -SqlAdmin $sqlAdmin `
  -SqlPasswordPlain $sqlPasswordPlain `
  -RunId $runId

az webapp restart `
  --resource-group $resourceGroup `
  --name $dispatcherApp

.\tools\runtime\Test-RuntimeNoOpSmokeState.ps1 `
  -SqlServer $sqlServer `
  -Database $db `
  -SqlAdmin $sqlAdmin `
  -SqlPasswordPlain $sqlPasswordPlain `
  -RunId $runId
```

## Expected state

The desired final state is `Completed`.

A failure is acceptable only when it is a real business/runtime failure from the smoke provider implementation, not one of these infrastructure/setup failures:

- missing run parent
- missing manifest provider
- missing mapping profile file
- invalid mapping profile JSON shape
- Service Bus message never consumed
