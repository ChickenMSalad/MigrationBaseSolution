# P7.9B SQL Operational Execution History Writer

## Purpose

P7.9B adds an application-level execution history writer and SQL-backed implementation for the P7 operational runtime.

This set intentionally does not change worker execution flow yet. It provides the service surface that the next runtime wiring set can use to record started, completed, and failed work-item execution attempts.

## Added files

- `src/Core/Migration.Application/Operational/ExecutionHistory/IOperationalExecutionHistoryWriter.cs`
- `src/Core/Migration.Infrastructure.Sql/Operational/ExecutionHistory/SqlOperationalExecutionHistoryOptions.cs`
- `src/Core/Migration.Infrastructure.Sql/Operational/ExecutionHistory/SqlOperationalExecutionHistoryWriter.cs`
- `src/Core/Migration.Infrastructure.Sql/Operational/ExecutionHistory/SqlOperationalExecutionHistoryServiceCollectionExtensions.cs`
- `scripts/Validate-P79BExecutionHistoryWriterPackage.ps1`

## Registration

The SQL worker host can register the writer with:

```csharp
builder.Services.AddSqlOperationalExecutionHistory(builder.Configuration);
```

Required using:

```csharp
using Migration.Infrastructure.Sql.Operational.ExecutionHistory;
```

## Configuration

The writer uses the existing operational store connection by default:

```json
{
  "ConnectionStrings": {
    "MigrationOperationalStore": "..."
  }
}
```

Optional override:

```json
{
  "SqlOperationalExecutionHistory": {
    "ConnectionStringName": "MigrationOperationalStore",
    "SchemaName": "migration",
    "ExecutionAttemptsTableName": "WorkItemExecutionAttempts"
  }
}
```

## Validation

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P79BExecutionHistoryWriterPackage.ps1
```

Then build:

```powershell
dotnet build
```
