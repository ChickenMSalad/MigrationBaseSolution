# P7.7A Runtime Source Capture

This set is intentionally repo-native and non-compiling. It captures the exact source surfaces needed before adding QueueExecutor SQL operational runtime wiring.

## Why this exists

P7.6 confirmed the correct runtime locations:

- `src\Workers\Migration.Workers.QueueExecutor`
- `src\Core\Migration.Infrastructure.Sql`
- `src\Core\Migration.Orchestration`
- `src\Core\Migration.GenericRuntime`
- `src\Hosts\Migration.Hosts.SqlOperationalWorker`

The next implementation set needs exact class names, method signatures, registration methods, option shapes, and existing loop behavior. This set generates that source capture from the local repo so the next implementation set can compile against the actual code instead of guessed APIs.

## What this adds

- `scripts\Validate-P7RuntimeSourceCapture.ps1`
- `scripts\Write-P7RuntimeSourceCapture.ps1`
- this README

## What this does not add

- no C# runtime code
- no project file changes
- no package changes
- no SQL changes
- no `src\Migration.Infrastructure`
- no `src\Migration.Worker`
- no `Migration.Infrastructure.Runtime`

## Run

From repo root:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P7RuntimeSourceCapture.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\Write-P7RuntimeSourceCapture.ps1
```

The generated report is written to:

```text
docs\p7\P7.7A-Runtime-Source-Capture.generated.md
```

Send that generated file before the next implementation set.

## Expected next set

After this source capture, the next implementation set should be:

```text
P7.7B QueueExecutor SQL Operational Runtime Wiring
```

That set should wire the existing QueueExecutor worker and SQL operational services together using exact repo signatures.
