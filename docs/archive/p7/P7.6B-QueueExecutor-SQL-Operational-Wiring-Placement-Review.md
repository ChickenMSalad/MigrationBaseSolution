# P7.6B - QueueExecutor SQL Operational Wiring Placement Review

This set intentionally does not add compile-time runtime code. It is a repo-native placement and surface review checkpoint before SQL queue execution code is added.

## Why this exists

Recent P7 attempts created invalid paths and namespaces. This set prevents repeating those mistakes before wiring SQL operational execution into the existing QueueExecutor runtime.

## Allowed locations for the next implementation set

- `src\Workers\Migration.Workers.QueueExecutor`
- `src\Core\Migration.Infrastructure.Sql`
- `src\Core\Migration.Orchestration`
- `src\Core\Migration.GenericRuntime`
- `src\Hosts\Migration.Hosts.SqlOperationalWorker`
- `database\sql\p7`
- `docs\p7`
- `scripts`

## Disallowed locations and namespaces

- Do not create `src\Migration.Infrastructure`.
- Do not create `src\Migration.Worker`.
- Do not reference or recreate `Migration.Infrastructure.Runtime`.
- Do not add inline `PackageReference Version` attributes.
- Do not create a parallel runtime framework.

## What to run

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P7QueueExecutorSqlWiringPlacement.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\Write-P7QueueExecutorSqlWiringSurfaceReview.ps1
```

The generated report will be written to:

```text
docs\p7\P7.6B-QueueExecutor-SQL-Wiring-Surface-Review.generated.md
```

Commit this review output before proceeding to the implementation set.
