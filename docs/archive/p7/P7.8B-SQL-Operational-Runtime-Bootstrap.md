# P7.8B SQL Operational Runtime Bootstrap

This set consolidates the SQL fixes discovered during local P7 worker smoke testing.

## What this set does

- Ensures `migration.Runs` exists as a real table, not a compatibility view.
- Preserves existing `migration.MigrationRuns` data by copying missing rows into `migration.Runs`.
- Ensures `migration.WorkItems` has the columns expected by the current QueueExecutor runtime.
- Ensures `migration.ManifestRows` has the columns expected by the current fan-out/runtime path.
- Adds a schema validation script for the operational runtime tables.

## What this set intentionally does not do

- It does not drop existing operational tables.
- It does not seed a smoke run.
- It does not require `SqlOperationalQueueExecutor:RunId` in secrets.
- It does not alter package references or project files.

## Apply order

From the repo root:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P78BPackage.ps1
```

Then apply the SQL:

```powershell
sqlcmd -S "(localdb)\MSSQLLocalDB" -d MigrationOperationalStore -E -i .\database\sql\p7\006_sql_operational_runtime_bootstrap_compatibility.sql
```

Then validate the SQL runtime schema:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Test-P7SqlOperationalRuntimeSchema.ps1 -ConnectionString "Server=(localdb)\MSSQLLocalDB;Database=MigrationOperationalStore;Trusted_Connection=True;TrustServerCertificate=True"
```

```powershell
dotnet user-secrets remove "SqlOperationalQueueExecutor:RunId" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
```

```powershell

```
