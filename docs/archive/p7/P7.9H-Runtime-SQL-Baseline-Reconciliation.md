# P7.9H Runtime SQL Baseline Reconciliation

## Purpose

P7.9H turns the runtime SQL cleanup into a repeatable reconciliation step instead of another live-fire patching session.

This set is intentionally read-only. It inventories the active runtime SQL objects, identifies legacy GUID-era objects that still exist, and validates whether the runtime schema has been canonicalized to the current contract.

## Canonical runtime contract

| Area | Canonical object | Required shape |
|---|---|---|
| Run parent | `migration.Runs` | `RunId uniqueidentifier` |
| Work queue | `migration.WorkItems` | `WorkItemId bigint identity`, `RunId uniqueidentifier`, `ManifestRowId bigint null` |
| Manifest rows | `migration.ManifestRows` | `ManifestRowId bigint identity` |
| Work-item FK | `migration.WorkItems.RunId` | references `migration.Runs.RunId` |
| Manifest FK | `migration.WorkItems.ManifestRowId` | references `migration.ManifestRows.ManifestRowId` |

## Legacy objects that must not be active runtime dependencies

The following objects may exist temporarily during transition, but active dispatcher/executor paths must not depend on them:

- `migration.MigrationRuns`
- `migration.MigrationWorkItems`
- `migration.MigrationManifestRows`
- `migration.MigrationManifestRecords`
- `migration.MigrationFailures`
- `migration.MigrationIdentifierMaps`

## Intended usage

Run diagnostics first:

```powershell
.\tools\runtime\Invoke-RuntimeSqlBaselineReconciliationDiagnostics.ps1 `
  -SqlServer $sqlServer `
  -Database $db `
  -SqlAdmin $sqlAdmin `
  -SqlPasswordPlain $sqlPasswordPlain
```

Run validator after canonicalization scripts have been applied:

```powershell
.\tools\runtime\Invoke-RuntimeSqlBaselineReconciliationValidator.ps1 `
  -SqlServer $sqlServer `
  -Database $db `
  -SqlAdmin $sqlAdmin `
  -SqlPasswordPlain $sqlPasswordPlain
```

## Non-goals

This set does not drop legacy tables and does not rewrite data. It provides a safe reconciliation boundary so destructive cleanup can be planned separately and reviewed.
