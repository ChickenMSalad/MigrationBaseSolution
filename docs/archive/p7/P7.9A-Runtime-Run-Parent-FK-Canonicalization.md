# P7.9A - Runtime Run Parent FK Canonicalization

## Purpose

This set fixes the specific runtime drift discovered during cloud smoke testing:

- `migration.WorkItems.ManifestRowId` already points to canonical bigint `migration.ManifestRows`.
- `migration.WorkItems.RunId` still points to legacy `migration.MigrationRuns`.
- Runtime enqueue/smoke flow was forced to seed both `migration.Runs` and `migration.MigrationRuns` to satisfy conflicting assumptions.

That mixed parentage is not handoffable. Runtime work items must use one canonical run table.

## Canonical decision

For the P7 bigint runtime path:

| Runtime concept | Canonical object |
|---|---|
| Run parent | `migration.Runs` |
| Work queue | `migration.WorkItems` |
| Manifest rows | `migration.ManifestRows` |
| Legacy GUID run table | `migration.MigrationRuns` remains legacy/compatibility only |

After this set, `migration.WorkItems.RunId` should reference `migration.Runs(RunId)`, not `migration.MigrationRuns(RunId)`.

## Apply sequence

From repo root:

```powershell
.\tools\validate-p7.9a-runtime-run-parent-fk.ps1
```

Run diagnostics first:

```powershell
.\tools\runtime\Invoke-RuntimeRunParentFkDiagnostics.ps1 `
  -SqlServer $sqlServer `
  -Database $db `
  -SqlAdmin $sqlAdmin `
  -SqlPasswordPlain $sqlPasswordPlain
```

Apply canonicalization:

```powershell
.\tools\runtime\Invoke-RuntimeRunParentFkCanonicalization.ps1 `
  -SqlServer $sqlServer `
  -Database $db `
  -SqlAdmin $sqlAdmin `
  -SqlPasswordPlain $sqlPasswordPlain
```

Validate:

```powershell
.\tools\runtime\Invoke-RuntimeRunParentFkValidator.ps1 `
  -SqlServer $sqlServer `
  -Database $db `
  -SqlAdmin $sqlAdmin `
  -SqlPasswordPlain $sqlPasswordPlain
```

## Safety notes

- This set does **not** drop legacy tables.
- This set backfills missing `migration.Runs` rows from `migration.MigrationRuns` before changing the FK.
- This set refuses to continue if `migration.Runs`, `migration.WorkItems`, or `migration.MigrationRuns` is missing.
- This set is idempotent.
