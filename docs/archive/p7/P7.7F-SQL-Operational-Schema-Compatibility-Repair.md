# P7.7F SQL Operational Schema Compatibility Repair

This is a forward-only repair for the partially applied P7 SQL operational schema.

## Why this exists

The SQL worker host now starts far enough to run operational readiness, but readiness expects these physical tables under the `migration` schema:

- `MigrationProjects`
- `MigrationRuns`
- `MigrationManifestRows`
- `MigrationWorkItems`
- `MigrationFailures`
- `MigrationRunCheckpoints`
- `MigrationAssetMappings`

The current database already has most operational tables, but it is missing several compatibility table names expected by the P7 runtime readiness service.

## Apply order from current state

Do not drop the database.

From repo root:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P7SqlOperationalSchemaCompatibility.ps1

sqlcmd -S "(localdb)\MSSQLLocalDB" -d MigrationOperationalStore -E -i .\database\sql\p7\005_sql_operational_schema_compatibility_repair.sql

powershell -ExecutionPolicy Bypass -File .\scripts\Test-P7SqlOperationalReadinessTables.ps1 -ConnectionString "Server=(localdb)\MSSQLLocalDB;Database=MigrationOperationalStore;Trusted_Connection=True;TrustServerCertificate=True"
```

Then continue the P7 SQL apply path:

```powershell
sqlcmd -S "(localdb)\MSSQLLocalDB" -d MigrationOperationalStore -E -i .\database\sql\p7\002_operational_queue_procedures.sql
sqlcmd -S "(localdb)\MSSQLLocalDB" -d MigrationOperationalStore -E -i .\database\sql\p7\003_operational_queue_runtime_wiring.sql
sqlcmd -S "(localdb)\MSSQLLocalDB" -d MigrationOperationalStore -E -i .\database\sql\p7\004_smoke_seed_operational_run.sql
```

Run diagnostics:

```powershell
sqlcmd -S "(localdb)\MSSQLLocalDB" -d MigrationOperationalStore -E -i .\database\sql\p7\004_sql_operational_smoke_diagnostics.sql
```

Run the worker:

```powershell
$env:DOTNET_ENVIRONMENT="Development"
dotnet run --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj *> .\worker-run.log
```

## Important

This set does not introduce C# runtime changes, package changes, or project changes.
