# P7.9G Runtime Validator Hardening

## Purpose

This set introduces a narrow, reusable script-quality gate for runtime cleanup/deployment tooling. It exists because earlier cleanup validators repeatedly failed in the same ways:

- validators scanned themselves and matched their own rule text;
- literal forbidden strings inside rule definitions caused false positives;
- PowerShell StrictMode surfaced unsafe direct XML property access;
- `$path:` style string interpolation caused parser/runtime failures;
- helper scripts accidentally printed command arguments that may contain secrets.

## Scope

This set is intentionally conservative. It does not mutate runtime code, SQL schema, app settings, project files, or cloud resources.

It adds:

- a focused quality checker for PowerShell scripts;
- a report generator for validator-hardening evidence;
- a sample baseline configuration;
- a validator for this set.

## Important guardrails

The quality checker requires explicit target paths. It does not scan the entire repository by default. This prevents unrelated historical scripts from failing a targeted cleanup set.

The set validator validates the runtime scripts introduced by this set, but it does not quality-scan itself. Validators may parse themselves in future sets, but they should not scan their own rule definitions as target script content.

## Intended usage

Run this against specific cleanup/runtime script paths, for example:

```powershell
.\tools\runtime\Test-RuntimePowerShellScriptQuality.ps1 `
  -Path .\tools\runtime
```

Generate an evidence report:

```powershell
.\tools\runtime\New-RuntimeValidatorHardeningReport.ps1 `
  -Path .\tools\runtime `
  -OutputPath .\artifacts\runtime-validator-hardening\report.md
```

Validate this drop-in set:

```powershell
.\tools\validate-p7.9g-runtime-validator-hardening.ps1
```

## What this set does not do

It does not declare the existing repo fully clean. It provides a safer reusable mechanism for future P7.9 cleanup sets to use without repeating the fragile self-reference mistakes.
