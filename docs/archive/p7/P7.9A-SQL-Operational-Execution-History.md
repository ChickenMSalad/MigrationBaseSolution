# P7.9A SQL Operational Execution History

This set adds a durable SQL execution-history surface for the operational worker runtime.

It is intentionally conservative:

- no runtime code changes
- no project file changes
- no package references
- no mutation of existing queue/worker behavior
- additive SQL objects only

## What this adds

- `migration.WorkItemExecutionAttempts`
- `migration.vw_WorkItemExecutionAttemptSummary`
- `migration.usp_RecordWorkItemExecutionAttemptStarted`
- `migration.usp_RecordWorkItemExecutionAttemptCompleted`

This creates the database surface needed for later worker instrumentation so each claimed work item can have a durable attempt record with start time, completion time, result JSON, error classification, retryability, and duration.

## Apply

```powershell
sqlcmd -S "(localdb)\MSSQLLocalDB" -d MigrationOperationalStore -E -i .\database\sql\p7\007_sql_operational_execution_history.sql
```

## Validate package

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P79AExecutionHistoryPackage.ps1
```

## Validate database schema

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Test-P7SqlOperationalExecutionHistorySchema.ps1 -ConnectionString "Server=(localdb)\MSSQLLocalDB;Database=MigrationOperationalStore;Trusted_Connection=True;TrustServerCertificate=True"
```

## Operational notes

P7.9A only establishes the SQL execution-history schema. The next implementation set can wire the worker so each claimed item calls the start/completion procedures around real execution.

This keeps the risk low after the P7.8 SQL/runtime stabilization work.
