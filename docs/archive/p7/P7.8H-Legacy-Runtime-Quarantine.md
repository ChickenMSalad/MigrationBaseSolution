# P7.8H Legacy Runtime Quarantine

## Purpose

This set quarantines old GUID-era operational runtime references so active dispatcher, executor, queue, and SQL operational worker paths do not drift back to legacy tables or legacy identifier types.

This set is intentionally non-destructive. It does not drop SQL objects and does not mutate runtime code. It adds inventory, allowlist, and boundary validation so legacy references are visible and contained.

## Canonical runtime contract

| Concept | Canonical runtime contract |
|---|---|
| Work item table | `migration.WorkItems` |
| Manifest row table | `migration.ManifestRows` |
| Work item id | SQL `bigint identity` / C# `long` |
| Manifest row id | SQL `bigint identity` / C# `long` / `long?` when optional |
| Run id | SQL `uniqueidentifier` / C# `Guid` |
| Service Bus work message WorkItemId | numeric value serialized from `long` |

## Quarantined legacy surface

The following objects and references are legacy unless a file is clearly historical, documentation-only, or an explicit compatibility/repair script:

- `dbo.MigrationWorkItems`
- `migration.MigrationWorkItems`
- `dbo.MigrationManifestRows`
- `migration.MigrationManifestRows`
- `migration.MigrationManifestRecords`
- GUID `WorkItemId` in active runtime paths
- GUID `ManifestRowId` / `ManifestRecordId` in active runtime paths

## Commands

Generate SQL object inventory:

```powershell
sqlcmd `
  -S "$sqlServer.database.windows.net" `
  -d $db `
  -U $sqlAdmin `
  -P $sqlPasswordPlain `
  -i .\database\sql\p7\011_legacy_runtime_object_inventory.sql
```

Generate repo legacy-reference inventory:

```powershell
.\tools\runtime\New-LegacyRuntimeReferenceInventory.ps1 `
  -RepoRoot . `
  -OutputPath .\artifacts\runtime\legacy-runtime-reference-inventory.csv
```

Validate active boundary:

```powershell
.\tools\runtime\Test-LegacyRuntimeReferenceBoundary.ps1 `
  -RepoRoot .
```

Validate this drop-in:

```powershell
.\tools\validate-p7.8h-legacy-runtime-quarantine.ps1
```

## Notes

This set exists because the live deployment exposed repeated drift between old GUID-era operational tables and the current bigint-backed SQL runtime. The boundary checks should fail before deployment if active runtime code points back at legacy table names.
