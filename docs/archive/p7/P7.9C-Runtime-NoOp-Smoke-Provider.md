# P7.9C - Runtime NoOp Smoke Provider

## Purpose

This set replaces the fake `Csv` smoke path with an explicit runtime smoke provider set.

The previous smoke flow proved the platform path worked up to `GenericMigrationJobRunner`, but then failed because the smoke job was pretending to be a real `Csv` manifest job and depended on a loose `smoke.json` mapping file. That is not a stable release gate.

This set adds first-class smoke runtime providers to the Service Bus executor host:

- `RuntimeSmokeManifestProvider`
- `RuntimeSmokeSourceConnector`
- `RuntimeSmokeTargetConnector`
- `RuntimeSmokeServiceCollectionExtensions`

The smoke manifest returns an empty manifest row list by default. That lets the runner validate runtime composition and finish without requiring real connector credentials, real manifest files, or real assets.

## Required Program.cs registration

The apply script inserts this registration into:

```text
src/Workers/Migration.Workers.ServiceBusExecutor/Program.cs
```

```csharp
using Migration.Workers.ServiceBusExecutor.Smoke;

builder.Services.AddRuntimeSmokeExecutionProviders();
```

The registration is intentionally local to the Service Bus executor host. It does not alter generic production provider registration and does not add another background worker.

## Required project file publish item

The apply script also ensures this file is published:

```text
src/Workers/Migration.Workers.ServiceBusExecutor/runtime-smoke.mapping.json
```

## Apply

```powershell
.\tools\runtime\Apply-RuntimeSmokeProvider.ps1
```

Then validate:

```powershell
.\tools\validate-p7.9c-runtime-noop-smoke-provider.ps1
```

Use:

  "targetType": "RuntimeSmoke",
  "manifestType": "RuntimeSmoke",
  "mappingProfilePath": "runtime-smoke.mapping.json",
  "dryRun": true,
