# P7.9F Clean Dev Runtime Rebuild Execution

## Purpose

This set turns the reset-readiness work into a controlled execution handoff. It does not delete or recreate cloud resources directly. It generates a dated rebuild command plan that an operator can inspect, edit, and run step-by-step.

## Why this exists

The runtime deployment failed previously because SQL, app settings, queue state, and published binaries drifted independently. The correct recovery path is not more live patching. The correct path is a deterministic clean rebuild sequence with gates before and after each destructive or deployment action.

## Canonical sequence

1. Export current Azure App Service settings.
2. Export current Azure SQL runtime schema.
3. Export Service Bus queue counts.
4. Run reset readiness SQL.
5. Generate reset command plan.
6. Review generated commands manually.
7. Apply canonical SQL baseline.
8. Apply canonical app settings.
9. Deploy dispatcher and executor from clean publish output.
10. Run NoOp smoke only, not fake Csv smoke.
11. Capture evidence.

## Safety rules

- The plan generator writes commands to a file; it does not run destructive commands.
- The evidence bundle copies/export-references existing local artifacts only.
- SQL validation is read-only.
- No script in this set uses XML mutation.
- No script should print secret values.
