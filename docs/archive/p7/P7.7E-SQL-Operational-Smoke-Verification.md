# P7.7E SQL Operational Smoke Verification

This set is a safe operational checkpoint after P7.7B/P7.7C/P7.7D.

It does **not** add runtime code, mutate project files, or execute migrations.

## Purpose

Before enabling `SqlOperationalMigrationJobExecutor:Enabled=true`, verify that the SQL operational store, SQL worker host, and no-op completion mode are configured safely.

## Adds

- `scripts/Validate-P7SqlOperationalSmokeVerification.ps1`
- `scripts/Write-P7SqlOperationalSchemaDiagnostic.ps1`
- `scripts/Show-P7SqlOperationalSmokeCommands.ps1`
- `database/sql/p7/004_sql_operational_smoke_diagnostics.sql`
- `docs/p7/P7.7E-SQL-Operational-Smoke-Verification.md`

## Does not add

- runtime C# files
- project references
- packages
- inline package versions
- new top-level `src/Migration.*` folders
- `Migration.Infrastructure.Runtime` namespaces

## Apply

Copy files into the repository root.

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P7SqlOperationalSmokeVerification.ps1
```

## Local secret posture

For safe smoke testing, keep real migration execution disabled:

```powershell
dotnet user-secrets set "SqlOperationalWorker:Enabled" "true" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "SqlOperationalWorker:CompleteNoOpWorkItems" "true" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "SqlOperationalMigrationJobExecutor:Enabled" "false" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
```

## Diagnostics

Generate the SQL operational schema diagnostic report:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Write-P7SqlOperationalSchemaDiagnostic.ps1 -ConnectionString "Server=localhost;Database=MigrationOperationalStore;Trusted_Connection=True;TrustServerCertificate=True"
```

The report writes:

```text
docs\p7\P7.7E-SQL-Operational-Schema-Diagnostic.generated.md
```

## Smoke run posture

Run the SQL worker host:

```powershell
dotnet run --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj --environment Development
```

Expected safe behavior:

- worker starts
- SQL connection opens
- readiness succeeds if scripts 001/002/003 are applied
- no-op SQL work items can be completed only when `CompleteNoOpWorkItems=true`
- real migration job execution remains blocked while `SqlOperationalMigrationJobExecutor:Enabled=false`
