# P7.10B - Canonical SQL Baseline Handoff

## Purpose

P7.10B converts the runtime SQL cleanup from a collection of historical scripts into a handoffable baseline sequence.

This set does not mutate SQL by itself. It documents and validates the canonical runtime SQL order after the P7.9 cleanup work.

## Canonical runtime SQL contract

The active SQL runtime contract is:

- `migration.Runs` is the canonical run parent table.
- `migration.WorkItems.RunId` must reference `migration.Runs.RunId`.
- `migration.WorkItems.WorkItemId` is a `bigint` identity key.
- `migration.ManifestRows.ManifestRowId` is a `bigint` identity key.
- `migration.WorkItems.ManifestRowId` references `migration.ManifestRows.ManifestRowId` when present.
- `migration.MigrationRuns`, `migration.MigrationWorkItems`, and `migration.MigrationManifestRecords` are legacy compatibility objects and must not be used by active Service Bus runtime workers.

## Canonical handoff order

Use this order for a clean dev runtime deployment:

1. Apply the current runtime schema scripts under `database/sql/p7`.
2. Apply P7.9A run-parent FK canonicalization.
3. Run the P7.9A FK validator.
4. Run the P7.9H SQL baseline reconciliation validator.
5. Run the P7.10B SQL baseline handoff validator.
6. Apply canonical App Service settings from P7.9B.
7. Deploy dispatcher and executor from a clean publish output.
8. Run P7.9D NoOp smoke.
9. Capture P7.9I evidence.

## What is intentionally not done here

This set does not delete legacy tables. Legacy table deletion should only happen after a dedicated data-retention decision and a separate destructive migration review.

This set does not introduce another smoke path. The only canonical runtime smoke path is `RuntimeSmoke` from P7.9C/P7.9D.

## Success criteria

P7.10B passes when:

- the handoff SQL inventory script identifies canonical and legacy runtime objects,
- the validator checks actual SQL metadata rather than fragile plain-text assumptions,
- PowerShell wrapper scripts are parser-safe and do not recursively quality-scan themselves,
- the documented handoff path points to the P7.9 canonical runtime flow.
