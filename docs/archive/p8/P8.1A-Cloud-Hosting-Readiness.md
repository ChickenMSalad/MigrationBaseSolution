# P8.1A Cloud Hosting Readiness

P8.1A starts the cloud-hosting phase without changing runtime behavior. It validates the P7 operational baseline and generates an inventory of host, worker, SQL runtime, execution-history, and configuration surfaces that should drive the P8.1 Azure hosting implementation.

## Scope

This set is intentionally read-only with respect to runtime behavior. It adds validation and inventory scripts only.

## Apply

From the repository root:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P81ACloudHostingReadiness.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\Write-P81ACloudHostingReadinessInventory.ps1
dotnet build
```

The inventory script writes:

```text
docs\p8\P8.1A-Cloud-Hosting-Readiness-Inventory.generated.md
```

## Expected baseline

The SQL operational worker host should run with no `SqlOperationalQueueExecutor:RunId` secret and should either process runnable SQL runs or log that no runnable migration runs were found.

Required local development secrets should stay minimal:

```json
{
  "ConnectionStrings:MigrationOperationalStore": "Server=(localdb)\\MSSQLLocalDB;Database=MigrationOperationalStore;Trusted_Connection=True;TrustServerCertificate=True",
  "SqlOperationalQueueExecutor:Enabled": "true",
  "SqlOperationalQueueExecutor:WorkerId": "local-sql-operational-worker-01",
  "SqlOperationalQueueExecutor:BatchSize": "10",
  "SqlOperationalQueueExecutor:LeaseSeconds": "300",
  "SqlOperationalQueueExecutor:PollDelaySeconds": "5",
  "SqlOperationalQueueExecutor:RunUntilIdleAndStop": "false",
  "SqlOperationalMigrationJobExecutor:Enabled": "false"
}
```

Do not configure `SqlOperationalQueueExecutor:RunId` except as a short-lived debug override.
