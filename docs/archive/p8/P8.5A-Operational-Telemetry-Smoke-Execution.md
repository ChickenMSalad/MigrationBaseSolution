# P8.5A Operational Telemetry Smoke Execution

This set moves from OpenTelemetry wiring to proof-of-life verification.

It does not mutate runtime code. It verifies that the current repo has the required telemetry surfaces and documents the smallest safe smoke path for local and Azure Monitor validation.

## Goal

Prove that operational execution emits observable traces from these runtime paths:

- SQL operational queue worker
- Service Bus dispatcher
- Service Bus executor

## Local no-exporter smoke configuration

Use this first. It verifies the OTEL registration path without requiring Azure Monitor ingestion.

```powershell
dotnet user-secrets set "OpenTelemetry:EnableTracing" "true" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "OpenTelemetry:EnableAzureMonitorExporter" "false" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "OpenTelemetry:TraceSamplingRatio" "1.0" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
```

Repeat for the Service Bus dispatcher/executor only when you are ready to run the distributed path.

## Azure Monitor smoke configuration

Use this after local startup is clean.

```powershell
dotnet user-secrets set "OpenTelemetry:EnableTracing" "true" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "OpenTelemetry:EnableAzureMonitorExporter" "true" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "OpenTelemetry:AzureMonitorConnectionString" "<APPLICATIONINSIGHTS_CONNECTION_STRING>" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
```

Alternative environment variables:

```powershell
$env:MIGRATION_OpenTelemetry__EnableTracing = "true"
$env:MIGRATION_OpenTelemetry__EnableAzureMonitorExporter = "true"
$env:MIGRATION_OpenTelemetry__TraceSamplingRatio = "1.0"
$env:APPLICATIONINSIGHTS_CONNECTION_STRING = "<APPLICATIONINSIGHTS_CONNECTION_STRING>"
```

## Minimal local smoke sequence

1. Build.

```powershell
dotnet build
```

2. Run SQL operational worker with no RunId override.

```powershell
dotnet run --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj --environment Development
```

3. Confirm startup logs still show readiness and either runnable-run processing or idle behavior.

Expected healthy behavior includes:

- `AddOperationalOpenTelemetry` path does not crash startup.
- readiness probe still passes.
- worker still discovers runnable runs without requiring `SqlOperationalQueueExecutor:RunId`.
- queue idle behavior is unchanged when no runnable work exists.

## Distributed Service Bus smoke sequence

Only run this after Service Bus connection settings are available.

1. Apply equivalent OpenTelemetry settings to:

- `src\Workers\Migration.Workers.ServiceBusDispatcher`
- `src\Workers\Migration.Workers.ServiceBusExecutor`

2. Start dispatcher.

```powershell
dotnet run --project .\src\Workers\Migration.Workers.ServiceBusDispatcher\Migration.Workers.ServiceBusDispatcher.csproj --environment Development
```

3. Start executor.

```powershell
dotnet run --project .\src\Workers\Migration.Workers.ServiceBusExecutor\Migration.Workers.ServiceBusExecutor.csproj --environment Development
```

4. Validate Service Bus send/receive behavior and SQL work-item completion/failure behavior.

## Azure Monitor verification

In Application Insights / Azure Monitor, look for traces with ActivitySource:

```text
Migration.Operational.Execution
```

Expected operation names:

- `SqlQueueWorkItemExecution`
- `ServiceBusDispatch`
- `ServiceBusWorkItemExecution`

Expected tags include:

- `migration.run.id`
- `migration.work_item.id`
- `messaging.servicebus.correlation_id`
- `messaging.message.id`
- `migration.execution.duration_ms`

## Do not change in this set

- Do not change queue settlement behavior.
- Do not change SQL schema.
- Do not reintroduce RunId cloud overrides.
- Do not add broad source mutation scripts.
