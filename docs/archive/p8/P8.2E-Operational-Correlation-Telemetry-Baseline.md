# P8.2E Operational Correlation / Telemetry Baseline

This set establishes the repo baseline for distributed execution correlation and structured telemetry without changing runtime worker behavior.

## Goals

- Standardize the correlation fields that must flow through SQL operational execution, Service Bus dispatch/execution, and execution history.
- Keep the Service Bus execution path aligned with the already-proven SQL operational execution boundary.
- Avoid creating a parallel telemetry framework.
- Prepare the next runtime patch to enrich logs/scopes safely.

## Required correlation fields

Use these names consistently in structured logs, generated docs, operational APIs, and future Activity tags:

- `RunId`
- `WorkItemId`
- `ManifestRowId`
- `WorkerId`
- `WorkItemType`
- `PartitionKey`
- `AttemptCount`
- `ServiceBusMessageId`
- `ServiceBusCorrelationId`
- `ExecutionAttemptId`
- `ExecutionDurationMs`
- `ExecutionOutcome`
- `ErrorCode`

## Boundary guidance

The preferred instrumentation boundary is the same execution boundary already used by the SQL and Service Bus workers:

- `ISqlOperationalWorkItemExecutor.ExecuteAsync`
- `SqlOperationalWorkItemWorker.ExecuteClaimedItemAsync`
- `SqlOperationalServiceBusWorkItemExecutor.ExecuteAsync`
- `SqlServiceBusExecutorWorker` settlement paths
- `IOperationalExecutionHistoryWriter.ExecuteAsync`

## Logging rules

- Prefer structured logging templates over interpolated strings.
- Include `RunId` and `WorkItemId` on every execution-path log event.
- Include Service Bus message metadata only in Service Bus-specific paths.
- Never log connector secrets, raw credentials, SAS tokens, bearer tokens, or full connection strings.
- Treat `PayloadJson` and `ResultJson` as sensitive-by-default; log size/hash/summaries only unless explicitly in local debug.

## P8.2F target

P8.2F should wire correlation scopes into the SQL operational worker and Service Bus executor paths using the existing operational execution boundary. It should not rewrite connector execution or introduce new package references unless compile errors prove they are required.
