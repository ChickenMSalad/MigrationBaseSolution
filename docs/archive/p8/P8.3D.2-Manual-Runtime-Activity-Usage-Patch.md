# P8.3D.2 Manual Runtime Activity Usage Patch

This set intentionally does not include source mutation/repair scripts.

Use this document to manually add OpenTelemetry Activity usage at the exact repo-native boundaries identified by `P8.3D.1-Runtime-Activity-Usage-Patch-Inventory.generated.md`.

## Goal

Add runtime Activity creation around these boundaries:

- SQL queue work item execution
- Service Bus work item execution
- Service Bus dispatch

Preserve all existing behavior:

- `BeginScope` logging scopes
- SQL completion/failure updates
- Service Bus complete/dead-letter/abandon behavior
- execution-history writes
- runnable-run discovery

## Required using

For each file that uses the helper, add:

```csharp
using Migration.Application.Operational.Telemetry;
```

## 1. SQL queue worker

File:

```text
src\Workers\Migration.Workers.QueueExecutor\Services\SqlOperationalWorkItemWorker.cs
```

Method:

```text
ExecuteClaimedItemAsync
```

The inventory showed these anchors:

```text
var startedAtUtc = DateTimeOffset.UtcNow;
using var telemetryScope = _logger.BeginScope(...);
var result = await _executor.ExecuteAsync(item, cancellationToken).ConfigureAwait(false);
```

Immediately after the existing `BeginScope`, add:

```csharp
using var activity = OperationalExecutionActivity.StartSqlQueueWorkItemExecution(
    runId: item.RunId,
    workItemId: item.WorkItemId,
    manifestRowId: item.ManifestRowId,
    workItemType: item.WorkItemType,
    attemptCount: item.AttemptCount,
    partitionKey: item.PartitionKey);
```

After the executor result is available, before completion/failure persistence, add:

```csharp
OperationalExecutionActivity.SetExecutionResult(
    activity,
    result.Succeeded,
    result.ErrorCode,
    result.ErrorMessage);

OperationalExecutionActivity.SetExecutionDuration(
    activity,
    startedAtUtc,
    DateTimeOffset.UtcNow);
```

For the catch path that already captures failure time, set the result/duration before SQL failure persistence when practical:

```csharp
OperationalExecutionActivity.SetExecutionResult(
    activity,
    false,
    "UNHANDLED_EXCEPTION",
    ex.Message);

OperationalExecutionActivity.SetExecutionDuration(
    activity,
    startedAtUtc,
    DateTimeOffset.UtcNow);
```

If the local variable names differ, preserve the existing names from the method.

## 2. Service Bus executor

File:

```text
src\Workers\Migration.Workers.ServiceBusExecutor\Runtime\SqlServiceBusExecutorWorker.cs
```

The inventory showed these anchors:

```text
using var telemetryScope = _logger.BeginScope(...);
var result = await _executor.ExecuteAsync(workItem, args.CancellationToken);
CompleteMessageAsync(...)
DeadLetterMessageAsync(...)
AbandonMessageAsync(...)
```

Immediately after the existing `BeginScope`, add:

```csharp
var activityStartedAtUtc = DateTimeOffset.UtcNow;
using var activity = OperationalExecutionActivity.StartServiceBusWorkItemExecution(
    runId: workItem.RunId,
    workItemId: workItem.WorkItemId,
    manifestRowId: workItem.ManifestRowId,
    workItemType: workItem.WorkItemType,
    attemptCount: workItem.AttemptCount,
    partitionKey: workItem.PartitionKey,
    serviceBusCorrelationId: args.Message.CorrelationId,
    serviceBusMessageId: args.Message.MessageId,
    serviceBusDeliveryCount: args.Message.DeliveryCount);
```

After the executor result is available, before SQL queue update or Service Bus settlement, add:

```csharp
OperationalExecutionActivity.SetExecutionResult(
    activity,
    result.Succeeded,
    result.ErrorCode,
    result.ErrorMessage);

OperationalExecutionActivity.SetExecutionDuration(
    activity,
    activityStartedAtUtc,
    DateTimeOffset.UtcNow);
```

Do not change settlement behavior.

## 3. Service Bus dispatcher

File:

```text
src\Workers\Migration.Workers.ServiceBusDispatcher\Dispatching\SqlWorkItemDispatcher.cs
```

The inventory showed these anchors:

```text
ServiceBusMessage message = new(json)
MessageId = item.WorkItemId.ToString("N")
CorrelationId = item.RunId.ToString("N")
await _sender!.SendMessageAsync(message, cancellationToken).ConfigureAwait(false);
await MarkDispatchedAsync(item.WorkItemId, cancellationToken).ConfigureAwait(false);
```

Immediately before sending the message, add:

```csharp
var activityStartedAtUtc = DateTimeOffset.UtcNow;
using var activity = OperationalExecutionActivity.StartServiceBusDispatch(
    runId: item.RunId,
    workItemId: item.WorkItemId,
    manifestRowId: item.ManifestRowId,
    workItemType: item.WorkItemType,
    attemptCount: item.AttemptCount,
    partitionKey: item.PartitionKey,
    serviceBusCorrelationId: message.CorrelationId,
    serviceBusMessageId: message.MessageId);
```

Immediately after the send succeeds, add:

```csharp
OperationalExecutionActivity.SetExecutionResult(activity, true, null, null);
OperationalExecutionActivity.SetExecutionDuration(activity, activityStartedAtUtc, DateTimeOffset.UtcNow);
```

If the send throws, the Activity will be disposed with exception context only if the method records it elsewhere. Do not add catch/rethrow unless there is already local error handling.
