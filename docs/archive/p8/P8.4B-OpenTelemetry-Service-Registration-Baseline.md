# P8.4B OpenTelemetry Service Registration Baseline

This set adds the repo-local configuration contract needed before wiring `services.AddOpenTelemetry()` into hosts.

It intentionally does not add OpenTelemetry packages or mutate host startup files. The next implementation set can add package references and host-specific calls after this baseline confirms the configuration surface is stable.

## Added source files

- `src/Core/Migration.Application/Operational/Telemetry/OperationalTelemetryRegistrationOptions.cs`
- `src/Core/Migration.Application/Operational/Telemetry/OperationalTelemetryConfigurationKeys.cs`

## Configuration model

The canonical section is:

```json
{
  "OpenTelemetry": {
    "Enabled": false,
    "ServiceName": "MigrationBaseSolution",
    "ServiceNamespace": "Migration",
    "ServiceInstanceId": "",
    "EnableTracing": true,
    "EnableMetrics": false,
    "EnableConsoleExporter": false,
    "EnableAzureMonitorExporter": false,
    "AzureMonitorConnectionString": "",
    "TraceSamplingRatio": 1.0
  }
}
```

For cloud deployment with the existing `MIGRATION_` configuration provider, use double underscores:

- `MIGRATION_OpenTelemetry__Enabled`
- `MIGRATION_OpenTelemetry__ServiceName`
- `MIGRATION_OpenTelemetry__ServiceNamespace`
- `MIGRATION_OpenTelemetry__EnableTracing`
- `MIGRATION_OpenTelemetry__EnableAzureMonitorExporter`
- `MIGRATION_OpenTelemetry__AzureMonitorConnectionString`

## Package plan for the next implementation set

Do not put inline versions on `PackageReference` items. If packages are added, update centralized package management only.

Likely package references for the eventual host registration set:

- `OpenTelemetry.Extensions.Hosting`
- `OpenTelemetry.Exporter.Console` for local diagnostics only
- `Azure.Monitor.OpenTelemetry.Exporter` for Azure Monitor export

The exact central versions should be verified against the repo's existing `Directory.Packages.props` before adding them.

## Runtime registration plan

The future registration extension should:

1. Bind `OperationalTelemetryRegistrationOptions` from the `OpenTelemetry` section.
2. Register `OperationalExecutionActivitySources.Name` with tracing.
3. Set service resource attributes from `ServiceName`, `ServiceNamespace`, and `ServiceInstanceId`.
4. Keep exporters disabled by default unless explicitly enabled.
5. Prefer Azure Monitor exporter only when a connection string is provided.
6. Preserve no-op behavior for local development.

## Why this set is staged

P8.3 introduced ActivitySource, Activity helpers, and runtime Activity usage. P8.4 should add exporter/runtime registration without destabilizing the worker hosts. This baseline separates the configuration contract from package/project mutation.
