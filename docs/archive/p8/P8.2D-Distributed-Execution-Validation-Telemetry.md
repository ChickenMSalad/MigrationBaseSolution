# P8.2D Distributed Execution Validation / Telemetry

This set validates the Service Bus distributed execution path after the P8.2C adapter work.

## Purpose

P8.2C replaced the placeholder Service Bus executor with an adapter over the SQL operational execution boundary. P8.2D does not change runtime behavior. It verifies the distributed execution topology and documents the expected end-to-end behavior.

## Expected runtime chain

1. SQL operational runtime owns durable run/work-item state.
2. Service Bus dispatcher publishes work-item messages from SQL.
3. Service Bus executor receives messages.
4. Executor resolves the SQL operational work item.
5. Executor invokes the SQL operational execution boundary.
6. Executor completes or fails the SQL work item.
7. Executor settles the Service Bus message.
8. Execution history records the attempt.
9. Dead-letter behavior remains reserved for invalid messages, missing work items, or non-retryable execution failures.

## Validation commands

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P82DDistributedExecutionValidationTelemetry.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\Write-P82DDistributedExecutionTelemetryInventory.ps1
dotnet build
```

## Operational expectations

- Service Bus executor should not use `PlaceholderServiceBusWorkItemExecutor`.
- Service Bus executor should use `SqlOperationalServiceBusWorkItemExecutor`.
- Service Bus executor should preserve existing `CompleteOperationalWorkItemRequest`, `FailOperationalWorkItemRequest`, and `DeadLetterMessageAsync` paths.
- Service Bus executor should keep `ServiceBusWorkItemMessage` as the message contract.
- Service Bus executor should use the same SQL operational execution boundary as the SQL queue executor.
- Service Bus dispatcher and executor should keep their existing worker-host shape.

## Required configuration themes

- `SqlServiceBusDispatcher:SqlConnectionString` or equivalent secure provider.
- `SqlServiceBusDispatcher:ServiceBusConnectionString` or managed identity equivalent.
- `SqlServiceBusDispatcher:QueueName`.
- `SqlServiceBusExecutor:ServiceBusConnectionString` or managed identity equivalent.
- `SqlServiceBusExecutor:QueueName`.
- SQL operational store connection for the runtime services.

Do not use `SqlOperationalQueueExecutor:RunId` in cloud/distributed execution.
