# P8.4A OpenTelemetry Runtime Registration Readiness

This set prepares the repo for actual OpenTelemetry/Azure Monitor runtime registration without mutating runtime code, adding package references, or changing project files.

## Purpose

P8.3 established the runtime telemetry foundation:

- canonical operational telemetry fields
- BeginScope correlation
- ActivitySource definitions
- Activity helper methods
- SQL queue execution Activity usage
- Service Bus dispatch/execution Activity usage
- Azure Monitor exporter readiness inventory

P8.4A validates that the repo is ready for runtime registration before adding exporter packages or host wiring.

## Required existing surfaces

- `OperationalExecutionActivitySources`
- `OperationalExecutionActivity`
- SQL queue worker Activity usage
- Service Bus executor Activity usage
- Service Bus dispatcher Activity usage
- existing `MIGRATION_` configuration provider usage
- existing Application Insights configuration awareness

## Next implementation target

P8.4B should add the first real OpenTelemetry runtime registration layer, likely as an additive extension under a repo-native hosting or infrastructure namespace. It should avoid host-specific copy/paste and expose one shared registration surface that workers and Admin API can call.

## Recommended configuration shape

Use environment variables compatible with existing `MIGRATION_` configuration binding:

- `MIGRATION_OpenTelemetry__Enabled`
- `MIGRATION_OpenTelemetry__ServiceName`
- `MIGRATION_OpenTelemetry__ServiceVersion`
- `MIGRATION_OpenTelemetry__EnableTracing`
- `MIGRATION_OpenTelemetry__EnableMetrics`
- `MIGRATION_OpenTelemetry__SamplingRatio`
- `MIGRATION_AzureMonitor__Enabled`
- `MIGRATION_AzureMonitor__ConnectionString`
- `APPLICATIONINSIGHTS_CONNECTION_STRING`

## Guardrails

- Do not add inline `Version` attributes to `PackageReference` items.
- Use central package management only.
- Do not mutate SQL schema.
- Do not rewrite worker execution loops.
- Preserve Service Bus settlement behavior.
- Preserve SQL completion/failure behavior.
- Preserve existing ActivitySource name: `Migration.Operational.Execution`.
