# P8.1G Operational Health Endpoint Surfaces

P8.1G adds reusable HTTP endpoint surfaces for the operational health probe contracts introduced in P8.1F.

## Added endpoint extension

Path:

```text
src\Core\Migration.Admin.Api\Endpoints\Operational\Health\OperationalHealthProbeEndpointExtensions.cs
```

Endpoints exposed by the extension:

```text
GET /health/live
GET /health/ready
```

Both endpoints return `OperationalHealthProbeResponse` from `IOperationalHealthProbeService`.

## Added service registration extension

Path:

```text
src\Core\Migration.Admin.Api\Registration\AdminApiOperationalHealthProbeRegistrationExtensions.cs
```

This extension registers:

```text
AddSqlOperationalRuntimeReadiness(configuration)
AddSqlOperationalHealthProbes()
```

## Required startup wiring

Wire services in the Admin API startup composition where other operational-store services are registered:

```csharp
builder.Services.AddAdminApiOperationalHealthProbes(builder.Configuration);
```

Wire endpoints where the Admin API operational endpoints are mapped:

```csharp
api.MapOperationalHealthProbeEndpoints();
```

Add the endpoint namespace if required:

```csharp
using Migration.Admin.Api.Endpoints.Operational.Health;
```

Add the registration namespace if required:

```csharp
using Migration.Admin.Api.Registration;
```

## Cloud probe contract

- Liveness: `/health/live`
- Readiness: `/health/ready`
- Idle worker with no runnable SQL runs remains healthy.
- `SqlOperationalQueueExecutor:RunId` remains debug-only and must not be required for readiness.

## Notes

This set does not convert the SQL operational worker host into an HTTP host. It exposes cloud probe surfaces through the existing Admin API host, which is the safest repo-native surface for HTTP endpoints.
