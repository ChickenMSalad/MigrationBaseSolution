# P8.3E Azure Monitor Exporter Readiness

This set prepares the repo for OpenTelemetry / Azure Monitor exporter wiring without changing runtime code, project files, package references, or SQL.

## Purpose

P8.3B and P8.3C introduced ActivitySource and activity helper foundations. P8.3D integrated Activity usage manually into runtime execution boundaries. P8.3E documents and validates the next boundary: exporter readiness.

## Current architecture expectation

The operational runtime should now have:

- canonical telemetry fields
- logger scope correlation
- ActivitySource constants
- runtime Activity helper methods
- Activity usage in SQL queue execution
- Activity usage in Service Bus execution
- Activity usage in Service Bus dispatch

Exporter wiring should be added later only after package compatibility and centralized package version management are confirmed.

## Required configuration shape

Production should prefer Azure Monitor / Application Insights connection string configuration through cloud settings or Key Vault-backed settings.

Supported setting names:

```text
ApplicationInsights:ConnectionString
APPLICATIONINSIGHTS_CONNECTION_STRING
AzureMonitor:ConnectionString
OpenTelemetry:Enabled
OpenTelemetry:ServiceName
OpenTelemetry:ServiceVersion
OpenTelemetry:Exporter
```

## Explicit non-goals

This set does not:

- add OpenTelemetry package references
- add Azure Monitor package references
- modify Directory.Packages.props
- modify worker source files
- modify Admin API startup
- mutate runtime behavior

## Next implementation after this set

P8.3F should inspect central package versions and add exporter registration only if compatible with the repo's package management conventions.
