# P8.2A Service Bus Execution Readiness

P8.2 starts the distributed execution phase. This set intentionally does not mutate runtime behavior.

## Purpose

This set inventories the existing repo-native Service Bus dispatcher/executor surfaces before adding distributed execution changes.

## Expected existing surfaces

- `src/Workers/Migration.Workers.ServiceBusDispatcher`
- `src/Workers/Migration.Workers.ServiceBusExecutor`
- `SqlServiceBusDispatcherWorker`
- `SqlServiceBusExecutorWorker`
- `SqlServiceBusExecutorOptions`
- `IServiceBusWorkItemExecutor`
- `ServiceBusWorkItemMessage`
- Azure host role defaults for Service Bus executor/dispatcher where present

## P8.2 direction

P8.2 should evolve the existing SQL-backed operational runtime into a distributed execution topology:

1. SQL operational runs remain the source of truth.
2. Dispatcher workers fan out runnable/leased SQL work into Service Bus messages.
3. Executor workers consume Service Bus messages and execute work items.
4. SQL remains authoritative for completion, failure, retry, execution history, and run coordination.
5. Service Bus provides scale-out transport, not the durable operational state.

## Non-goals for this set

- No package changes.
- No SQL changes.
- No project-file changes.
- No Service Bus runtime mutation.
- No cloud resource deployment.
