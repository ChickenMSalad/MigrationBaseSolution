# P8.5B Azure Monitor Proof-of-Life

This set moves from telemetry wiring to operational proof-of-life without mutating runtime code.

## Goal

Prove that the operational runtime can emit OpenTelemetry Activities and export them to Azure Monitor / Application Insights from the three runtime hosts:

- `Migration.Hosts.SqlOperationalWorker`
- `Migration.Workers.ServiceBusDispatcher`
- `Migration.Workers.ServiceBusExecutor`

## Preconditions

- P8.4C runtime registration is applied.
- P8.5A smoke execution readiness is committed.
- `dotnet build` passes.
- The runtime hosts contain `AddOperationalOpenTelemetry(builder.Configuration)`.
- `OpenTelemetry:EnableTracing` is enabled for the host being tested.

## Local no-exporter proof

Use this when you want to verify runtime stability without sending traces to Azure Monitor.

```powershell
dotnet user-secrets set "OpenTelemetry:EnableTracing" "true" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "OpenTelemetry:EnableAzureMonitorExporter" "false" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "OpenTelemetry:TraceSamplingRatio" "1.0" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
```

Then run the SQL operational worker as usual.

## Azure Monitor proof

Set an Application Insights connection string in one of these forms:

```powershell
dotnet user-secrets set "OpenTelemetry:EnableTracing" "true" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "OpenTelemetry:EnableAzureMonitorExporter" "true" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
dotnet user-secrets set "OpenTelemetry:AzureMonitorConnectionString" "<connection-string>" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
```

or use the environment variable:

```powershell
$env:APPLICATIONINSIGHTS_CONNECTION_STRING = "<connection-string>"
```

## Expected telemetry names

The ActivitySource name should be:

```text
Migration.Operational.Execution
```

Expected Activity names:

```text
SqlQueueWorkItemExecution
ServiceBusDispatch
ServiceBusWorkItemExecution
```

## Expected tags

Look for tags such as:

```text
migration.run.id
migration.work_item.id
messaging.servicebus.correlation_id
messaging.message.id
migration.execution.duration_ms
migration.execution.succeeded
migration.execution.error_code
```

## Smoke-test approach

1. Start with `Migration.Hosts.SqlOperationalWorker` only.
2. Confirm the worker boots with OTEL enabled.
3. Confirm no exporter error occurs when exporter is disabled.
4. Enable Azure Monitor exporter with a valid connection string.
5. Run a small SQL operational smoke run.
6. Confirm traces appear in Azure Monitor / Application Insights.
7. Add dispatcher and executor after the SQL worker is proven.

## Do not do yet

- Do not enable partial sampling during first proof.
- Do not wire every legacy host.
- Do not introduce another telemetry abstraction layer.
- Do not change execution semantics to satisfy tracing.
