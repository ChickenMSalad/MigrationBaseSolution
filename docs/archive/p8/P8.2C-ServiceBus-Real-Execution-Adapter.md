# P8.2C Service Bus Real Execution Adapter

P8.2C replaces the Service Bus executor placeholder path with an adapter that delegates to the existing SQL operational execution boundary.

## Intent

The Service Bus executor already owns message settlement and SQL completion/failure updates. The missing piece was replacing `PlaceholderServiceBusWorkItemExecutor` with an adapter that invokes `ISqlOperationalWorkItemExecutor`.

## Added

- `SqlOperationalServiceBusWorkItemExecutor`
  - fetches the SQL operational work item by `WorkItemId`
  - invokes `ISqlOperationalWorkItemExecutor.ExecuteAsync`
  - maps the SQL result to `ServiceBusWorkItemExecutionResult`

## Wiring

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Repair-P82CWireServiceBusRealExecutionAdapter.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P82CServiceBusRealExecutionAdapter.ps1
dotnet build
```

## Boundaries Preserved

- `SqlServiceBusExecutorWorker` still owns Service Bus settlement.
- SQL completion and failure calls remain in the executor worker.
- Dead-letter behavior remains in the executor worker.
- `ISqlOperationalWorkItemExecutor` remains the real execution boundary.
- No package references are added.
