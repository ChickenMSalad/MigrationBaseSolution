# P7.10H - Post-Clean Baseline Review

## Purpose

This set records the state after the P7.9/P7.10 cleanup sequence and defines the remaining work that should happen after the clean runtime baseline has been validated.

This set is intentionally non-mutating. It does not change runtime code, SQL schema, app settings, or deployment behavior.

## Clean baseline facts to preserve

- Runtime smoke should use `RuntimeSmoke`, not the old Csv smoke path.
- Runtime worker app settings should use canonical unprefixed keys for SQL, Service Bus, dispatcher, executor, and operational queue settings.
- `migration.WorkItems` must reference the canonical runtime run parent, not the legacy GUID-era parent path.
- Dispatcher and executor should be deployable from clean publish output.
- NoOp runtime smoke should complete end-to-end before tagging or handing off a deployment.

## Remaining cleanup categories

1. **Legacy object retirement**
   - Keep legacy GUID-era objects quarantined until migration history requirements are clear.
   - Do not delete legacy objects without a dedicated migration and backup plan.

2. **Local/cloud parity**
   - Rebuild or refresh local DB using the canonical P7.9/P7.10 baseline path.
   - Compare canonical runtime tables, foreign keys, and required indexes.

3. **CI enforcement**
   - Add a CI workflow only after local gates are stable.
   - CI should run build plus targeted validators only.

4. **Operational evidence**
   - Keep evidence bundles for the clean baseline smoke run.
   - Record app settings snapshots after cleanup.

5. **Future runtime work**
   - Expand smoke from 0-row NoOp to small deterministic manifest execution.
   - Add integration tests around Service Bus dispatch/execution state transitions.

## Out of scope

- Deleting legacy control-plane files.
- Removing legitimate Csv migration support.
- Wiping cloud resources.
- Rewriting active runtime worker logic.
