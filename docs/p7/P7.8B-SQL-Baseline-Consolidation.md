# P7.8B SQL Baseline Consolidation

## Purpose

This set turns the P7 SQL runtime into a handoffable deployment surface instead of a collection of ad hoc repair scripts.

It does **not** drop or rewrite existing runtime tables. It adds validation, deployment-order documentation, and scripts that prove local and Azure are using the same SQL contract before anyone deploys worker code.

## Why this exists

The cloud smoke uncovered several avoidable contract mismatches:

- The deployed runtime used `migration.WorkItems` with `bigint` identities, while older SQL/code paths still referenced `MigrationWorkItems` and GUID IDs.
- Some procedures/indexes assumed `IX_WorkItems_ClaimQueue` existed before the actual cloud table had it.
- The worker code and SQL scripts disagreed on columns such as `LeaseOwner`, `LeaseExpiresUtc`, `CreatedUtc`, `UpdatedUtc`, and `WorkItemType`.
- Local and Azure databases drifted badly enough that successful builds provided no meaningful deployment safety.

## Canonical runtime SQL contract

The SQL operational runtime contract for Service Bus dispatcher/executor is:

| Object | Role | Canonical status |
|---|---|---|
| `migration.WorkItems` | Durable runtime work queue | Active |
| `migration.ManifestRows` | Runtime manifest row table | Active |
| `migration.Runs` | Runtime coordinator run table | Active |
| `migration.WorkItemExecutionAttempts` | Attempt history | Active |
| `migration.WorkItemFailures` | Failure history | Active |
| `migration.WorkerHeartbeats` | Worker status | Active |
| `migration.MigrationWorkItems` | Legacy GUID-era operational table | Legacy / not used by workers |
| `migration.MigrationManifestRecords` | Legacy GUID-era manifest table | Legacy / not used by workers |

ID contract:

| Field | Type |
|---|---|
| `RunId` | `uniqueidentifier` / `Guid` |
| `WorkItemId` | `bigint` / `long` |
| `ManifestRowId` | `bigint` / `long?` |
| `FailureId` and legacy correlation IDs | `uniqueidentifier` / `Guid` |

## Deployment order

For a clean P7 runtime database, use this order:

1. `database/sql/p7/006_sql_operational_runtime_bootstrap_compatibility.sql`
2. `database/sql/p7/007_sql_operational_execution_history.sql`
3. `database/sql/p7/009_runtime_sql_contract_validator.sql`

Historical scripts under `database/sql/operational` and earlier P7 scripts remain in the repo for now, but worker deployments should be gated by the validator above.

## Local/Azure parity rule

Before deploying dispatcher or executor code:

1. Export local runtime schema.
2. Export Azure runtime schema.
3. Compare the exports.
4. Run `009_runtime_sql_contract_validator.sql` against the target database.
5. Only deploy if the validator passes.

## What this set adds

```text
database/sql/p7/009_runtime_sql_contract_validator.sql
tools/runtime/Export-RuntimeSqlSchema.ps1
tools/runtime/Compare-RuntimeSqlSchema.ps1
tools/runtime/Invoke-RuntimeSqlContractValidator.ps1
tools/validate-p7.8b-sql-baseline-consolidation.ps1
```

## Non-goals

This set does not:

- Delete legacy GUID tables.
- Rewrite existing deployment scripts.
- Change worker code.
- Change app settings.
- Seed smoke data.

Those belong in later sets once the contract is guarded.
