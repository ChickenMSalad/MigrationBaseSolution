# P7.10B P7 Closeout Readiness

This set is intentionally non-mutating for runtime code and SQL schema. It adds a conservative closeout validator and a generated closeout checklist.

## Files

- `scripts/Validate-P710BP7CloseoutReadiness.ps1`
- `scripts/Write-P710BP7CloseoutChecklist.ps1`
- `docs/p7/P7.10B-P7-Closeout-Readme.md`

## Apply

Copy the files into the repository root.

## Validate

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P710BP7CloseoutReadiness.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\Write-P710BP7CloseoutChecklist.ps1
dotnet build
```

## Runtime smoke

Remove any pinned RunId secret before the final P7 closeout smoke:

```powershell
dotnet user-secrets remove "SqlOperationalQueueExecutor:RunId" --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj
```

Run the SQL operational worker:

```powershell
dotnet run --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj --environment Development
```

Expected idle state when no runnable runs exist:

```text
RunIdOverride=(null)
SQL operational worker host readiness check passed. Status=Ready
SQL operational queue idle. No runnable migration runs found.
```
