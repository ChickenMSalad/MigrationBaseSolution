# P7.10C - Local/Azure Runtime Parity

## Purpose

This set verifies that a local SQL/runtime environment and the Azure dev runtime environment agree on the canonical runtime contract after the P7.9 cleanup work.

This set is intentionally non-mutating. It exports evidence, compares the canonical runtime object surface, and writes a parity report.

## Canonical runtime surface

The runtime baseline is considered aligned when both environments expose the following active objects and configuration concepts:

- `migration.Runs`
- `migration.WorkItems`
- `migration.ManifestRows`
- `ConnectionStrings__MigrationOperationalStore`
- `SqlServiceBusDispatcher__*`
- `SqlServiceBusExecutor__*`
- `SqlOperationalWorkItemQueue__*`
- `SqlOperationalMigrationJobExecutor__Enabled`

Legacy GUID-era tables can exist only as quarantined/historical objects. Active runtime FKs should point to `migration.Runs` and `migration.ManifestRows`.

## Expected flow

1. Export local SQL object snapshot.
2. Export Azure SQL object snapshot.
3. Compare both snapshots.
4. Generate a parity report.
5. Commit the evidence if it is useful for handoff.
