# P7.10E - Stale Runtime Artifact Quarantine

## Purpose

P7.10E adds a non-destructive inventory and quarantine plan for stale runtime artifacts that caused confusion during the P7.8/P7.9 deployment cleanup.

This set does not delete files. It produces a report so stale artifacts can be reviewed, archived, or removed in a separate explicit commit.

## Why this exists

The runtime is now validated through the canonical path:

- `migration.Runs`
- `migration.WorkItems`
- `migration.ManifestRows`
- `RuntimeSmoke` NoOp provider
- canonical unprefixed runtime app settings

Historical control-plane exports, old Csv smoke definitions, previous deployment evidence, and generated runtime parity files should not be treated as active runtime configuration.

## Included files

- `config-samples/runtime-stale-artifact-quarantine.sample.json`
- `tools/runtime/New-RuntimeStaleArtifactInventory.ps1`
- `tools/runtime/New-RuntimeStaleArtifactQuarantineReport.ps1`
- `tools/validate-p7.10e-stale-runtime-artifact-quarantine.ps1`

## Usage

From repo root:

```powershell
.\tools\validate-p7.10e-stale-runtime-artifact-quarantine.ps1
```

Generate an inventory report:

```powershell
.\tools\runtime\New-RuntimeStaleArtifactQuarantineReport.ps1 `
  -ConfigurationPath .\config-samples\runtime-stale-artifact-quarantine.sample.json `
  -OutputPath .\artifacts\runtime-cleanup\stale-artifact-quarantine-report.md
```

## Rules

The report is intentionally advisory. It should identify likely stale files but must not delete or mutate anything.

Review the report before making any cleanup commits.
