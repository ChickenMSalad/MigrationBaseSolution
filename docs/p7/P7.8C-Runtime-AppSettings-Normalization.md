# P7.8C Runtime AppSettings Normalization

## Purpose

This set prevents a repeat of the P7.7/P7.8 cloud smoke deployment issues where worker hosts used a mixture of old and new appsetting names. The goal is to make Service Bus dispatcher/executor configuration explicit, reviewable, and safe to clean up.

## Canonical convention

Runtime worker hosts should use unprefixed .NET configuration keys unless a host explicitly proves it binds a different section.

Canonical keys:

```text
ConnectionStrings__MigrationOperationalStore
SqlOperationalRuntimeReadiness__ConnectionString
OpenTelemetry__EnableTracing
OpenTelemetry__EnableAzureMonitorExporter
SqlServiceBusDispatcher__Enabled
SqlServiceBusDispatcher__SqlConnectionString
SqlServiceBusDispatcher__ServiceBusConnectionString
SqlServiceBusDispatcher__QueueName
SqlServiceBusExecutor__ServiceBusConnectionString
SqlServiceBusExecutor__QueueName
SqlOperationalMigrationJobExecutor__Enabled
SqlOperationalWorkItemQueue__SchemaName
SqlOperationalWorkItemQueue__WorkItemsTableName
GenericMigrationRuntime__RegisterAllWhenEmpty
GenericMigrationRuntime__EnabledManifests__0
GenericMigrationRuntime__EnabledSources__0
GenericMigrationRuntime__EnabledTargets__0
```

## Known stale or duplicate families

These names existed during the cloud smoke debugging and should not be treated as canonical runtime keys:

```text
MIGRATION_ConnectionStrings__MigrationOperationStore
MIGRATION_ServiceBusDispatcher__*
MIGRATION_ServiceBusExecutor__*
MIGRATION_SqlWorkItemDispatcher__*
MIGRATION_SqlServiceBusExecutor__*
MIGRATION_SqlOperationalWorkItemQueue__*
OperationalStore__Sql__*
```

Do not delete them blindly. Generate a cleanup plan, confirm the host code no longer binds them, then delete in a separate deployment step.

## Safe workflow

1. Export current appsettings from Azure.
2. Generate a cleanup plan from the exported JSON.
3. Apply canonical settings.
4. Restart the app.
5. Run the runtime smoke test.
6. Only after a clean smoke test, delete stale keys.

## Example

```powershell
az webapp config appsettings list --resource-group $resourceGroup --name $dispatcherApp -o json > .\dispatcher-appsettings.json
az webapp config appsettings list --resource-group $resourceGroup --name $executorApp -o json > .\executor-appsettings.json

.\tools\runtime\Get-RuntimeAppSettingsCleanupPlan.ps1 -Path .\dispatcher-appsettings.json -Role Dispatcher
.\tools\runtime\Get-RuntimeAppSettingsCleanupPlan.ps1 -Path .\executor-appsettings.json -Role Executor
```

## Acceptance criteria

- Dispatcher app has only the dispatcher canonical Service Bus keys plus shared connection/telemetry keys.
- Executor app has only executor canonical Service Bus keys, job executor key, work item queue table keys, generic runtime keys, and shared connection/telemetry keys.
- No host depends on `MIGRATION_*` duplicate keys for the Service Bus runtime path.
- No host depends on `OperationalStore__Sql__*` to override the SQL queue table.
