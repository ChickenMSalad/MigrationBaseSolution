# P7.9I Runtime Deployment Evidence Handoff

## Purpose

This set creates a handoffable deployment evidence workflow after the runtime SQL baseline and configuration cleanup work.

It does not mutate Azure resources, SQL schema, app settings, or source code. It only provides templates and scripts for collecting evidence that a deployment was performed from a clean repo state and that the dispatcher/executor runtime can be evaluated consistently.

## Why this exists

The previous runtime work exposed several failure modes:

- Cloud and local SQL drift.
- GUID-era and bigint runtime objects coexisting.
- Duplicated app setting families.
- Smoke scripts that used fake provider names or missing files.
- PowerShell wrapper scripts that could hang or fail under Windows PowerShell 5.1.
- Validators that self-scanned their own rule text.

This handoff set gives operators one place to record the evidence needed before declaring a dev runtime deployment usable.

## Files

- `config-samples/runtime-deployment-evidence.template.json`
- `tools/runtime/New-RuntimeDeploymentEvidenceBundle.ps1`
- `tools/runtime/Test-RuntimeDeploymentEvidenceBundle.ps1`
- `tools/validate-p7.9i-runtime-deployment-evidence-handoff.ps1`

## Expected evidence inputs

The evidence bundle can reference files produced by prior sets:

- Runtime environment snapshot.
- Runtime parity report.
- SQL FK validator output.
- SQL baseline reconciliation validator output.
- App settings canonicalization report.
- Service Bus queue state output.
- Dispatcher log excerpt.
- Executor log excerpt.
- Latest `migration.WorkItems` state query output.

## Operator rule

Do not call a deployment complete because the build succeeded. The deployment is complete only when the evidence bundle is present and the validator says the required evidence is available.
