# P7.8G Runtime Release Gates

## Purpose

This set adds a repo-local release gate wrapper for the SQL + Service Bus runtime cleanup effort.
It does not mutate runtime code, Azure resources, or SQL schema. It centralizes the checks introduced by P7.8A through P7.8F so a deployment can be blocked before Azure is touched.

## Why this exists

The cloud smoke exposed avoidable drift across:

- SQL schema and code types (`Guid` versus `long` for runtime work items and manifest rows)
- old GUID-era operational tables versus new bigint runtime tables
- duplicated and stale app settings
- dispatcher and executor deployment verification
- smoke payload shape versus registered runtime providers

A build alone is not a deployment gate. The gate must verify runtime contracts, settings, SQL schema, and smoke prerequisites.

## Intended usage

From the repo root:

```powershell
.\tools\runtime\Invoke-RuntimeReleaseGate.ps1 `
  -ConfigurationPath .\config-samples\runtime-release-gates.sample.json `
  -OutputPath .\artifacts\runtime-release-gate
```

Use the generated report as a pre-deploy handoff artifact.

## Gate philosophy

The gate should fail closed. Missing validators, missing appsetting snapshots, missing SQL schema exports, or dirty repo state should all be treated as non-deployable unless explicitly skipped.

This set intentionally produces report files instead of changing cloud settings or database objects.
