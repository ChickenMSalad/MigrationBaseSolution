# P7.9E Execution History Runtime Recording

## Purpose

P7.9E wires SQL operational execution-history recording into the QueueExecutor runtime boundary.

The integration point is intentionally narrow: `SqlOperationalWorkItemWorker.ExecuteClaimedItemAsync` records an execution attempt around `ISqlOperationalWorkItemExecutor.ExecuteAsync`.

## What this set changes

- Adds `IOperationalExecutionHistoryWriter` to `SqlOperationalWorkItemWorker`.
- Records execution attempt start before calling the work-item executor.
- Records completion after successful queue completion.
- Records retryable or terminal failure after queue failure handling.
- Registers SQL execution-history services from `AddSqlOperationalQueueExecutor`.

## What this set does not change

- No package references.
- No central package version changes.
- No SQL schema changes beyond requiring the already committed P7.9A execution-history SQL.
- No connector behavior changes.
- No admin API behavior changes.

## Required local secrets

Keep the P7.10 baseline only:

```json
{
  "ConnectionStrings:MigrationOperationalStore": "Server=(localdb)\\MSSQLLocalDB;Database=MigrationOperationalStore;Trusted_Connection=True;TrustServerCertificate=True",
  "SqlOperationalQueueExecutor:Enabled": "true",
  "SqlOperationalQueueExecutor:WorkerId": "local-sql-operational-worker-01",
  "SqlOperationalQueueExecutor:BatchSize": "10",
  "SqlOperationalQueueExecutor:LeaseSeconds": "300",
  "SqlOperationalQueueExecutor:PollDelaySeconds": "5",
  "SqlOperationalQueueExecutor:RunUntilIdleAndStop": "false",
  "SqlOperationalMigrationJobExecutor:Enabled": "false"
}
```

Do not configure `SqlOperationalQueueExecutor:RunId` except as a short-lived debug override.

## Apply and validate

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-P79EExecutionHistoryRuntimeRecording.ps1
dotnet build
```

Then run the worker host:

```powershell
dotnet run --project .\src\Hosts\Migration.Hosts.SqlOperationalWorker\Migration.Hosts.SqlOperationalWorker.csproj --environment Development
```

Expected baseline with no runnable runs:

```text
SQL operational QueueExecutor worker starting ... RunIdOverride=(null)
SQL operational worker host readiness check passed. Status=Ready
SQL operational queue idle. No runnable migration runs found.
```

## Operational note

Execution history writer failures are logged as warnings and do not block work-item execution. This keeps the worker operational even if history recording has a transient issue.
