# P7.8F Runtime Code Contract Guardrails

## Purpose

P7.8F adds repo-side guardrails so the SQL-backed runtime contract cannot silently regress back to the old GUID-era operational model.

This set is intentionally conservative. It does not change runtime behavior. It adds audit and validation tooling only.

## Canonical runtime contract

Active SQL Service Bus runtime code must treat these identifiers as follows:

| Concept | Canonical type | Owner |
|---|---:|---|
| `RunId` | `Guid` / `uniqueidentifier` | application/runtime correlation |
| `WorkItemId` | `long` / `bigint identity` | SQL runtime table |
| `ManifestRowId` | `long?` / `bigint identity` | SQL runtime table |
| `ManifestRecordId` in legacy mirror/old operational APIs | transitional/legacy | not part of the SQL Service Bus runtime queue contract |
| `FailureId`, `CheckpointId` | `Guid` where existing models require it | application correlation |

The active SQL Service Bus runtime queue table is:

```text
migration.WorkItems
```

The active runtime manifest row table is:

```text
migration.ManifestRows
```

The legacy GUID-era operational tables must not be referenced by active dispatcher/executor/runtime code:

```text
dbo.MigrationWorkItems
migration.MigrationWorkItems
dbo.MigrationManifestRows
migration.MigrationManifestRecords
```

## How to use

From repo root:

```powershell
.\tools\validate-p7.8f-runtime-code-contract-guardrails.ps1
```

Generate an inventory CSV:

```powershell
.\tools\runtime\New-RuntimeCodeContractInventory.ps1 `
  -OutputPath .\artifacts\runtime-code-contract-inventory.csv
```

Run the contract scan:

```powershell
.\tools\runtime\Test-RuntimeCodeContract.ps1
```

Run with an allowlist:

```powershell
.\tools\runtime\Test-RuntimeCodeContract.ps1 `
  -AllowListPath .\config-samples\runtime-code-contract.allowlist.sample.json
```

## Guardrail philosophy

This tool is not intended to delete old docs, old SQL, or legacy operational code in one sweep. It is intended to stop active runtime paths from accidentally using them.

Default scanned roots are:

```text
src
Workers
config-samples
```

The scanner excludes high-noise historical/generated/documentation locations by default:

```text
bin
obj
artifacts
docs
database/sql/operational
database/sql/cloud/output.txt
*.generated.md
```

## Expected follow-up

After this set, P7.8G should focus on removing or quarantining stale legacy SQL entry points from handoff documentation so a new deployer does not accidentally apply GUID-era operational scripts.
