# P7.10D CI Runtime Release Gate

## Purpose

This set adds a conservative CI/release-gate entry point for the cleaned runtime path. It is intentionally repo/local validation only. It does not deploy, mutate Azure, change SQL, purge queues, or run live smoke tests.

The gate is meant to prevent a repeat of the deployment drift issues found during P7.8/P7.9:

- GUID/bigint runtime contract drift
- stale SQL FK parentage
- stale `MIGRATION_*` runtime settings
- broken PowerShell validators
- self-scanning validators
- fake Csv/smoke.json runtime smoke paths

## Files

- `config-samples/runtime-ci-release-gate.sample.json`
- `tools/runtime/Invoke-RuntimeCiReleaseGate.ps1`
- `.github/workflows/runtime-ci-release-gate.sample.yml`
- `tools/validate-p7.10d-ci-runtime-release-gate.ps1`

## Recommended usage

Run locally from repo root:

```powershell
.\tools\validate-p7.10d-ci-runtime-release-gate.ps1

.\tools\runtime\Invoke-RuntimeCiReleaseGate.ps1 `
  -ConfigurationPath .\config-samples\runtime-ci-release-gate.sample.json `
  -OutputPath .\artifacts\runtime-ci-release-gate
```

The command writes a markdown report under `artifacts/runtime-ci-release-gate`.

## Notes

The workflow file is a sample only. Copy or adapt it when the repo is ready for CI wiring. It runs the same local gate and does not require secrets.
