# P7.9J Runtime Cleanup Closeout Gate

## Purpose

P7.9J provides a narrow closeout gate for the P7.9 runtime cleanup work. It does not mutate Azure resources, SQL databases, project files, or app settings. It only runs known targeted validators and writes a local evidence report.

This set exists to prevent the cleanup sequence from drifting back into ad hoc manual verification.

## Scope

The gate checks the P7.9 cleanup validators that are present in the working tree:

- P7.9A run-parent foreign key canonicalization
- P7.9B app settings canonicalization
- P7.9C runtime NoOp smoke provider
- P7.9D NoOp smoke harness
- P7.9E dev cloud reset readiness
- P7.9F clean dev runtime rebuild execution
- P7.9G validator hardening
- P7.9H SQL baseline reconciliation
- P7.9I deployment evidence handoff

Missing validators are reported as skipped, not failed, so the gate can be used while sets are still being introduced on a branch.

## Non-goals

This set does not:

- delete legacy SQL objects
- delete app settings
- deploy workers
- run Service Bus smoke tests
- inspect every PowerShell file in the repository
- enforce broad legacy cleanup outside P7.9

## Usage

From the repository root:

```powershell
.\tools\validate-p7.9j-runtime-cleanup-closeout.ps1
```

To write a report to a specific path:

```powershell
.\tools\runtime\Invoke-RuntimeCleanupCloseoutGate.ps1 `
  -OutputPath .\artifacts\runtime-cleanup-closeout\p7.9-closeout-report.md
```

## Success criteria

The gate passes when every present P7.9 validator exits successfully.

The report should be committed only if it is intended as evidence. In most cases, evidence files under `artifacts` should remain uncommitted.
