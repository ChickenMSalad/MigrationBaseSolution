# P4.14 Manifest Import Workspace

P4.14 adds the first operator-facing manifest intake workspace to the React Admin UI.

## Added files

| Path | Purpose |
|---|---|
| `apps/migration-admin-ui/src/lib/manifestImportApi.ts` | Client wrapper for the manifest import endpoint. |
| `apps/migration-admin-ui/src/components/ManifestImportPanel.tsx` | Operator panel for project ID, mapping profile ID, file selection, validation, and import submission. |

## Runtime endpoint

The UI posts multipart form data to:

```text
POST /api/operational/manifests/import
```

Expected fields:

| Field | Required | Notes |
|---|---:|---|
| `file` | Yes | CSV/XLSX manifest artifact. |
| `projectId` | Yes | Target migration project. |
| `importMode` | Yes | `validate-only` or `import`. |
| `mappingProfileId` | No | Optional mapping profile binding. |

## Design notes

This set intentionally keeps the UI thin. SQL remains the durable operational store, while CSV/Excel remain ingestion/export artifacts.
