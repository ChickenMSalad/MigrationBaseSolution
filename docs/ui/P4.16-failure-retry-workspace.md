# P4.16 Failure Retry Workspace

This set adds an operator-facing failure review and retry workflow to the React admin UI.

## Added files

```text
apps/migration-admin-ui/src/lib/failureRetryApi.ts
apps/migration-admin-ui/src/components/FailureRetryWorkspace.tsx
docs/ui/P4.16-failure-retry-workspace.md
```

## Intended Admin API contracts

```text
GET  /api/operational/runs/{runId}/failures?includeNonRetryable=false&limit=50&searchText=...
POST /api/operational/runs/{runId}/failures/retry
```

## Runtime purpose

P4.16 adds the first recovery operator workflow after run launch:

1. Search failures for a specific run.
2. Filter by asset, failure code, or message text.
3. Select retryable failure records.
4. Submit selected retries or all retryable failures.
5. Preserve operator notes for audit/recovery context.

The SQL operational store remains the durable source of truth. This set is UI/client-side wiring only and does not replace backend queue, run coordinator, or lease behavior.
