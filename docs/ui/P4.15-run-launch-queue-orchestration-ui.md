# P4.15 Run Launch + Queue Orchestration UI

This set adds the first operator-facing run launch workflow to the React admin UI.

## Added files

```text
apps/migration-admin-ui/src/lib/runLaunchApi.ts
apps/migration-admin-ui/src/components/RunLaunchPanel.tsx
docs/ui/P4.15-run-launch-queue-orchestration-ui.md
```

## Intended Admin API contracts

```text
POST /api/operational/runs/launch
GET  /api/operational/runs/{runId}/progress
POST /api/operational/runs/{runId}/cancel
```

The UI intentionally keeps the request model simple and operator-oriented:

```text
projectId
manifestImportId
mappingProfileId
executionMode
queueFanOutMode
requestedConcurrency
notes
```

## Runtime purpose

P4.15 connects the previously added pieces into an operator flow:

1. Select a project and optional manifest import.
2. Choose dry-run or full-run mode.
3. Choose queue fan-out behavior.
4. Launch the SQL-backed run.
5. Poll progress from the Admin API.
6. Allow cancellation signaling.

This is UI/client-side wiring only. Backend endpoint shapes can continue to mature behind the same URL contracts.
