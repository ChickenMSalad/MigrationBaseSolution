# P4.8 — SQL Service Bus Runtime Azure Scaffold

## Purpose

P4.8 establishes the first deployable Azure substrate for the SQL-first, Service-Bus-backed migration runtime.

It does not introduce new application contracts and does not mutate existing projects. The goal is to create the cloud resources needed by the already committed P4.1–P4.7 runtime layers.

## Resources

| Capability | Azure Resource |
|---|---|
| Durable operational state | Azure SQL |
| Work item dispatch | Azure Service Bus |
| Artifact staging | Azure Storage |
| Secrets | Azure Key Vault |
| Central configuration | Azure App Configuration |
| Metrics/logging | Application Insights + Log Analytics |
| Future workers | Container Apps Environment |

## Why this is separate

The repo already has deployment scaffolding areas under `deploy/azure`. P4.8 adds a dedicated SQL + Service Bus runtime slice rather than changing existing app-service, storage, key-vault, or container-app job scaffolds.
