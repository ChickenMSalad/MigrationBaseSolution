# P4.19 Connector Credential Vault + Secret References

P4.19 introduces operator-facing connector credential reference scaffolding.

The set intentionally stores references to secret material rather than secrets themselves. The intended cloud path is Azure Key Vault plus managed identity from the Admin API and worker execution environments.

Added areas:

- Admin API credential reference endpoints
- UI credential vault workspace
- application-level credential reference contracts
- SQL infrastructure placeholder store
- operator validation flow for connector credential references

This set is additive and does not replace existing connector configuration behavior.
