# P4.31 — P4 Runtime Stabilization Checks

This set adds validation scripts for the P4 runtime/control-plane work.

## What it validates

- `Program.cs` uses the consolidated operational endpoint composition.
- `App.tsx` uses the consolidated operational UI workspace composition.
- UI feature components are not directly duplicated in `App.tsx`.
- `.csproj` files do not use inline `PackageReference Version` attributes.
- no `payload` folder remains in the repository root after applying a drop-in.
- P4.30 local developer run profile files exist.

## Commands

From repo root:

```powershell
./scripts/validation/P4.31-Test-P4RuntimeStabilization.ps1
./scripts/validation/P4.31-Test-P4RuntimeBuild.ps1
```

To skip the UI build:

```powershell
./scripts/validation/P4.31-Test-P4RuntimeBuild.ps1 -SkipUiBuild
```

## Why this exists

The P4 sequence added a lot of runtime, API, and UI surface. Before adding more features, this gives the repo a repeatable health check that catches the issues that already occurred during P4:

- duplicate imports
- duplicate workspace JSX
- Program.cs growth
- inline package versions
- leftover payload folders
- build drift
