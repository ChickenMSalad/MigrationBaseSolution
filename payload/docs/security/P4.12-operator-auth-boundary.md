# P4.12 Operator Authentication Boundary

This set adds the first explicit operator authentication boundary scaffold for the SQL-backed migration control plane.

## Scope

- Admin API security status endpoint
- UI-side operator auth configuration model
- UI-side session snapshot helpers
- sample Entra ID configuration
- no tenant-specific secrets
- no forced authentication enforcement yet

## Runtime intent

The intended production flow is:

```text
migration-admin-ui
  -> obtains operator token from Entra ID
  -> calls Migration.Admin.Api with bearer token
  -> Migration.Admin.Api validates token, roles, and claims
  -> SQL-backed orchestration services execute authorized operations
```

## Endpoint

```text
GET /api/admin/security/status
```

The endpoint is intentionally diagnostic/readiness-oriented. It exposes whether Admin API auth configuration is present and whether the current request has an authenticated principal.

## Follow-up

A later set should enforce authorization policies on mutation endpoints after the operator role model is finalized.
