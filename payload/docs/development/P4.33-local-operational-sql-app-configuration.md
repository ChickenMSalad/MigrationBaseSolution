# P4.33 — Local Operational SQL App Configuration

This set documents how to point `Migration.Admin.Api` at the local operational SQL database created by P4.32.

## 1. Initialize local SQL

```powershell
./scripts/sql/P4.32-Initialize-OperationalSqlDatabase.ps1 -Apply
./scripts/sql/P4.32-Test-OperationalSqlDatabase.ps1
```

## 2. Copy sample settings

Copy:

```text
config-samples/appsettings.AdminApi.LocalOperationalSql.sample.json
```

into the Admin API project as a local-only settings file, for example:

```text
src/Core/Migration.Admin.Api/appsettings.LocalOperationalSql.json
```

Do not commit local secrets or machine-specific connection strings.

## 3. Run Admin API

```powershell
dotnet run --project src/Core/Migration.Admin.Api/Migration.Admin.Api.csproj --urls "https://localhost:55436"
```

## 4. Run UI

```powershell
cd apps/migration-admin-ui
npm run dev -- --host localhost --port 5174
```

## Expected keys

The local SQL runtime expects:

```json
{
  "ConnectionStrings": {
    "OperationalSql": "..."
  },
  "OperationalSql": {
    "ConnectionStringName": "OperationalSql"
  }
}
```

This keeps connection strings externalized and compatible with future Key Vault/App Configuration promotion.
